# 特征预处理（Feature Preprocessing)

## 第一部分：为什么需要特征预处理

### 1.1 问题背景

> 特征的**单位或者大小相差较大**，或者某特征的方差相比其他的特征要大出几个数量级，容易**影响（支配）目标结果**，使得一些模型（算法）无法学习到其他的特征。

**举例**：

- 特征 A：房屋面积（范围 50~500 平方米）
- 特征 B：卧室数量（范围 1~5 个）
- 特征 C：距地铁站范围（范围 0.1~10 公里，数值 0.1~10）
如果不做处理，距离（以公里计）的数值变化（0~10）可能远小于面积的变化（50~500），或者反过来，取决于量纲。实际计算欧氏距离时，数值大的特征会主导距离，导致其他特征几乎不起作用。

---

### 1.2 本质

特征预处理是将原始特征值通过某种数学变换，映射到**相同的尺度范围**或**相同的分布**，消除量纲和数量级差异，使每个特征在模型训练中具有相近的影响力。

---

### 1.3 哪些算法需要特征预处理

| 算法类型                               | 是否需要             | 原因                   |
| ---------------------------------- | ---------------- | -------------------- |
| KNN、K-Means、SVM、线性回归、逻辑回归、PCA、神经网络 | 必须               | 基于距离/梯度，特征尺度影响极大     |
| 决策树、随机森林、梯度提升树（GBDT）               | 通常不需要            | 基于分裂规则（阈值比较），尺度不影响   |
| 朴素贝叶斯                              | 不需要（若用高斯 NB 则可选） | 基于概率，但若特征方差差异大可考虑标准化 |

---

## 第二部分：归一化（Normalization）

### 2.1 定义与公式

> 通过对原始数据进行变换，把数据映射到 $[mi,mx]$ （默认为[0，1]）之间。

**公式**（Min-Max 归一化）：

$$x^{\prime}=x-\frac{\min\left(x\right)}{\max\left(x\right)-\min\left(x\right)}\times\left(mx-mi\right)+mi$$

其中：

- $x$：原始特征值
- $\min\left(x\right)$：该特征列的最小值
- $\max\left(x\right)$：该特征列的最大值
- $\left\lbrack\operatorname{\mathrm{mi}},mx\right\rbrack$：目标区间，默认 $mi=0,mx=1$

当目标区间为 $\left\lbrack0,1\right\rbrack$，公式简化为：

$$x^{\prime}=x-\frac{\min\left(x\right)}{\max\left(x\right)-\min\left(x\right)}$$

---

### 2.2 几何意义

将原始数据线性压缩到指定区间，保持原始数据的**相对顺序**和**比例关系**（但比例关系仅在原始数据经过线性变换后保持不变）。

---

### 2.3 归一化的 API

```python
from sklearn.preprocessing import MinMaxScaler

def dm01_MinMaxScaler():
    data = [[90, 2, 10, 40],
            [60, 4, 15, 45],
            [75, 3, 13, 46]]
    transformer = MinMaxScaler()          # 默认 feature_range=(0,1)
    data = transformer.fit_transform(data)
    print(data)
```

**参数详解**：

- `feature_range`：tuple，默认 `(0,1)`，指定缩放后的最小值、最大值。
- `copy`：bool，默认 True，是否复制数据。

**常用方法**：

| 方法                      | 说明                     |
| ----------------------- | ---------------------- |
| `fit(x)`                | 计算 x 中每列的最小值和最大值       |
| `transform(x)`          | 使用已计算的 min/max 进行缩放    |
| `fit_transform(x)`      | 先 fit 再 transform，一步完成 |
| `inverse_transform(x) ` | 逆变换，将归一化后的数据还原         |

---

### 2.4 手动计算示例

原始数据某一列：`[90,60,75]`

最小值=60，最大值=90，范围=30

归一化到\[0,1]：

- $90\to\frac{\left(90-60\right)}{30}=1.0$
- $60\to\frac{\left(60-60\right)}{30}=0.0$
- $75\to\frac{\left(75-60\right)}{30}=0.5$

---

### 2.5 归一化的优缺点

| 优点                                 | 缺点                                          |
| ---------------------------------- | ------------------------------------------- |
| 简单快速，结果固定在\[0,1]区间                 | **对异常值极其敏感**：若数据中有极端大或极端小的值，会压缩大部分正常数据到狭窄区间 |
| 保留了原始数据的分布形态（线性变换）                 | 需要知道每列的最大最小值，若新数据超出原范围，需重新 fit              |
| 适用于需要固定边界的场景（如图像像素 0-255 归一化到 0-1） | 不适合含有离群点的数据集                                |

> 如果出现异常点，影响了最大值和最小值，那么结果显然会发生改变，**鲁棒性较差，只适合传统精确小数据场景**。

---

## 第三部分：标准化（Standardization）

### 3.1 定义与公式

> 通过对原始数据进行标准化，转换为**均值为 0、标准差为 1**的标准正态分布的数据。

**公式**（Z-Score 标准化）：

$$\frac{x^{\prime}=x-\mu}{\sigma}$$

其中：

- $\mu$：该特征列的均值（mean）
- $\sigma$：该特征列的标准差（standard deviation）

**标准差计算**：

$$\sigma=\sqrt{\frac{1}{N}\sum_{i=1}^{N}\left(x_1-\mu\right)^2}$$

---

### 3.2 正态分布知识补充

正太分布曲线和 $3\sigma$ 法则：

- 约 68%的数据落在 $\left\lbrack\mu-\sigma,\mu+\sigma\right\rbrack$
- 约 95%的数据落在 $\left\lbrack\mu-2\sigma,\mu+2\sigma\right\rbrack$
- 约 99.7%的数据落在 $\left\lbrack\mu-3\sigma,\mu+3\sigma\right\rbrack$

标准化后的数据：

- 均值=0，标准差=1
- 数据范围大约在-3 到+3 之间（但理论上无界）

---

### 3.3 标准化的 API

```python
from sklearn.preprocessing import StandardScaler

def dm03_StandardScaler():
    data = [[90, 2, 10, 40],
            [60, 4, 15, 45],
            [75, 3, 13, 46]]
    transformer = StandardScaler()
    data = transformer.fit_transform(data)
    print(data)
    # 打印每列的均值和标准差
    print('transfer.mean_-->', transformer.mean_)
    print('transfer.var_-->', transformer.var_)
```

**属性**：

- `mean_`：每列训练时的均值（形状（n_features,））
- `var_`：每列训练时的方差（注意：实际 API 中为 `scale_` 或 `var_`，在 StandardScaler 中，`scale_` =标准差，`var_` =方差。）

**常用方法**：

| 方法                      | 说明                 |
| ----------------------- | ------------------ |
| `fit(x)`                | 计算 x 中每列的均值和标准差    |
| `transform(x)`          | 使用已计算的 均值和标准差进行标准化 |
| `fit_transform(x)`      | 先 fit 再 transform  |
| `inverse_transform(x) ` | 还原（乘以标准差再加均值）      |

---

### 3.4 手动计算示例

某一列：`[90,60,75]`

均值： $\mu=\frac{\left(90+60+75\right)}{3}=\frac{225}{3}=75$

方差：$=\frac{\left\lbrack\left(90-75\right)^2+\left(60-75\right)^2+\left(75-75\right)^2\right\rbrack}{3}=\frac{\left(225+225+0\right)}{3}=\frac{450}{3}=150$

标准差：$\sigma=\sqrt{150}\approx12.247$

标准化后：

- $90\to\frac{\left(90-75\right)}{12.247}\approx1.2247$
- $60\to\frac{\left(60-75\right)}{12.247}\approx-1.2247$
- $75\to\frac{\left(75-75\right)}{12.247}=0.0$

---

### 3.5 标准化的优缺点

| 优点                                      | 缺点                   |
| --------------------------------------- | -------------------- |
| **对异常值不敏感**（因为适用均值和标准差，少量异常影响有限）        | 不保证数据落在固定区间          |
| 适合大数据场景，鲁棒性好                            | 需要计算均值和标准差，计算量略大于归一化 |
| 不需求数据有边界，新数据超出原范围也可通过 $\mu ,\sigma$ 标准化 | 如果数据分布严重偏斜，标准化后仍偏斜   |

> 如果出现异常点，由于具有一定数据量，少量的异常点对于平均值的影响并不大。**适合现代嘈杂大数据场景**。

---

## 第四部分：归一化与标准化的对比

### 4.1 核心区别表

|维度|归一化（MinMaxScaler）|标准化（StandardScaler）|
|---|---|---|
|输出范围|固定区间（默认 [0,1]）|无固定范围，均值 0，标准差 1|
|对异常值鲁棒性|**差**（易受最大/最小值影响）|**好**（受均值/标准差影响小）|
|数据分布形态|保持原始形状（线性压缩）|改变为近似标准正态|
|适用场景|小规模、精确、无异常值的数据|大规模、嘈杂、可能有异常值的数据|
|是否保留 0 值|是（若 min=0，则最小值对应 0）|否（0 变为负值）|
|对稀疏数据的友好度|差（会破坏稀疏结构）|较好（可保留零中心）|
|是否需要知道范围|需要 min/max，若新数据超出则需重算|需要μ/σ，新数据可直接用|

---

### 4.2 选择建议

1. **优先使用标准化**，大多数情况下效果更好。
2. 只有在需要将数据严格限制在某一区间（如图神经网络输出层用 sigmoid 时，输入需在 0~1）时才用归一化。
3. 如果数据分布已知且无异常值，归一化也可用。
4. 对于图像像素（0 ~255),通常归一化到 0~1 或 1~1。

---

### 4.3 极端情况分析

**异常值影响示例**：

- 原始数据：`[1,2,3,4,100]`
- 归一化：最小值=1，最大值=100，$1\to0,2\to0.01,3\to0.02,4\to0.03,100\to1$。大部分数据被压缩到 0~0.03 之间，失去区分度。
- 标准化：均值 $=\frac{\left(1+2+3+4+100\right)}{5}=22$，标准差 $\approx39.7\to1\approx-0.53,2\approx-0.50,3\approx-0.48,4\approx-0.45,100\approx1.96$。正常数据仍分散在-0.5 附近，保留区分。

> 在**样本数量较大**的情况下，异常值对样本的均值和标准差的影响可以忽略不计。度量值 $X^{\prime}$ 容易受到样本中**最大值最小值**的影响，鲁棒性差。

## 第五部分：特征预处理在建模流程中的位置

$获取数据\to数据基本处理（缺失值、异常值）\to特征工程（包括预处理）\to模型训练$

具体到 KNN 鸢尾花案例中：

```python
# 划分训练集和测试集
X_train, X_test, y_train, y_test = train_test_split(...)

# 标准化
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)   # 对训练集拟合并转换
X_test = scaler.transform(X_test)         # 用训练集的均值和标准差转换测试集

# 训练模型
knn = KNeighborsClassifier()
knn.fit(X_train, y_train)
```

**重要原则**：

- **只对训练集 fit**，然后用训练集的参数（$\min/ max或\mu/\sigma$）去转换测试集。
- 绝对不能用测试集的信息去 fit，否则会造成数据泄露（测试集信息被模型间接学习）。

## 第六部分：归一化/标准化代码注释版

### 6.1 归一化完整示例

```python
import numpy as np
from sklearn.preprocessing import MinMaxScaler
data = np.array([[90, 2, 10, 40],
                 [60, 4, 15, 45],
                 [75, 3, 13, 46]])
# 归一化到默认 [0,1]
scaler = MinMaxScaler()
data_scaled = scaler.fit_transform(data)
print("归一化后:\n", data_scaled)
# 逆变换
data_original = scaler.inverse_transform(data_scaled)
# 归一化到 [-1, 1]
scaler2 = MinMaxScaler(feature_range=(-1, 1))
data_scaled2 = scaler2.fit_transform(data)
```

---

### 6.2 标准化完整示例

```python
from sklearn.preprocessing import StandardScaler

data = np.array([[90, 2, 10, 40],
                 [60, 4, 15, 45],
                 [75, 3, 13, 46]])

scaler = StandardScaler()
data_scaled = scaler.fit_transform(data)
print("标准化后:\n", data_scaled)
print("均值:", scaler.mean_)
print("标准差:", scaler.scale_)      # 注意：API中标准差属性为 scale_
print("方差:", scaler.var_)          # 若版本支持，否则可手动计算

# 验证：标准化后的数据均值为0，标准差为1
print("新数据的均值:", data_scaled.mean(axis=0))
print("新数据的标准差:", data_scaled.std(axis=0))
```

## 第七部分：常见问题与陷阱

### 7.1 是否需要先划分数据集再标准化？

- **必须**：先划分训练集和测试集，再对训练集 fit，对测试集 transform。
- 错误做法：对整个数据集标准化后再划分
	- 导致测试集信息泄露

---

### 7.2 分类特征需要标准化吗？

分类特征（如性别、颜色）通常不需要标准化，应使用独热编码（One-Hot Encoding）或其他编码方式。标准化适用于连续数值特征。

---

### 7.3 缺失值处理顺序？

先处理缺失值（删除或填充），再进行标准化/归一化。否则缺失值会导致预处理失败。

---

### 7.4 标准化后的数据范围是固定的吗？

不是。标准化只保证均值 0、标准差 1，但具体数值范围依赖于原始数据的分布。

例如：如果原始数据有极端值，标准化后可能出现-10 或+10 等。

---

### 7.5 关于鸢尾花案例的标准化使用

```python
# 3 数据集预处理 - 数据标准化
transfer = StandardScaler()
x_train = transfer.fit_transform(x_train)   # 拟合+转换训练集
x_test = transfer.transform(x_test)         # 只用转换测试集
```

**解析**：

`fit_transform` 对训练集完成拟合和转换，`transform` 只转化测试集。必须保持一致，否则测试集用的均值和标准差与训练集不同，模型评估无效。

## 第八部分：扩展知识

### 8.1 其他预处理方法

| 方法                    | 公式                                             | 用途                  |
| --------------------- | ---------------------------------------------- | ------------------- |
| 鲁棒缩放（RobustScaler）    | $z-\frac{\operatorname{\mathrm{median}}}{IQR}$ | 对异常值极其鲁棒（用中位数和四分位数） |
| 最大绝对值缩放（MaxAbsScaler） | $\frac{x}{\max\left(\vert x\vert\right)}$      | 稀疏数据保留 0 值          |
| 单位向量归一化（Normalizer）   | $\frac{x}{\vert x\vert_2}$                     | 将每个样本缩放到单位范数（行缩放）   |

---

### 8.2 何时使用 MinMaxScaler 而不是 StandardScaler？

- 需要将数据映射到固定区间（如图像、神经网络特定激活函数）。
- 数据的分布已知且没有极端异常值。
- 算法要求输入在 $\left\lbrack0,1\right\rbrack$ 或 $\left\lbrack-1,1\right\rbrack$（如某些核函数）。

---

### 8.3 何时使用 StandardScaler？

- 大多数机器学习算法（特别是线性模型、KNN、SVM、PCA）。
- 数据有异常值（但数量相对较少）。
- 不需要固定边界。

---

## 第九部分：总结

**归一化**：  
`x' = (x - min) / (max - min)`  
特点：固定区间 $\left\lbrack0,1\right\rbrack$，受极值影响大，小数据可用。

**标准化**：  
`x' = (x - μ) / σ`  
特点：均值 0 标准差 1，鲁棒性好，大数据推荐。

**常用 API**：

- 归一化：`MinMaxScaler()`
    
- 标准化：`StandardScaler()`
    
- 核心方法：`fit_transform(X_train)`，`transform(X_test)`

> 标准化适合现代嘈杂大数据场景。
