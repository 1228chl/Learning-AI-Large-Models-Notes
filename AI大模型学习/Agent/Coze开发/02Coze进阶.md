# Coze 平台超详细学习笔记（进阶篇）

## 一、私有数据访问

### 1.1 私有数据访问概述

在企业应用中，经常需要让大模型基于公司内部的私有数据（如产品手册、内部文档、知识库）来回答问题，而不是依赖其通用知识。这带来了两个核心挑战：

**数据安全挑战：** 私有数据包含商业机密，绝对不能上传到公共大模型进行处理，存在严重的泄露风险。

**时效性挑战：** 通用大模型的训练数据存在滞后性（如停更于 2023 年），无法获取企业最新的内部政策或产品信息。

**Coze 平台的解决方案：** Coze 平台提供了多种安全的私有数据访问方式，其中最主流、最高效的方案是采用 RAG（检索增强生成）技术。

### 1.2 RAG 知识库技术详解

RAG（Retrieval-Augmented Generation，检索增强生成）是目前解决大模型访问私有数据最有效、最流行的工程架构，通过“检索+生成”的模式结合外部知识库。

RAG 的工作流程包含四个核心步骤：

**第一步：数据准备（Data Prep）**
将私有文档（如 PDF、Word）进行清洗与切分，把长文本分割成语义完整的小文本块（Chunk），便于后续处理。

**第二步：向量化（Embedding）**
利用 Embedding 模型将文本块转化为高维向量，捕捉语义信息，然后将这些向量数据持久化存储在专用的向量数据库中。

**第三步：检索（Retrieval）**
用户提问后，系统将问题同样向量化，在向量库中通过相似度算法（如余弦相似度）查找与问题最相关的 Top-N 个文本块。

**第四步：生成（Generation）**
将检索到的相关文本块作为上下文，与用户问题拼接成 Prompt，发送给大模型，引导模型基于私有知识库生成准确回答。

### 1.3 Coze 平台 RAG 知识库操作步骤

Coze 平台内置了强大的 RAG 能力，无需编写复杂代码即可轻松搭建具备私有知识库问答能力的智能体系统。核心操作步骤如下：

**步骤一：创建知识库**
在平台资源库中，新建一个专属的知识库空间，用于存储私有文档。

**步骤二：上传私有文件**
将准备好的文档（支持 PDF、TXT 等主流格式）上传至新建的知识库中。

**步骤三：配置智能体关联**
进入智能体的设置界面，在知识库选项中勾选并关联刚刚创建的私有知识库。

**步骤四：自动生效完成部署**
Coze 会自动完成文档的向量化与索引构建，智能体即刻具备基于私有数据的问答能力。

### 1.4 数据库连接功能

除了知识库，Coze 还提供了数据库功能，适用于组织和管理结构化数据，例如客户信息、产品列表、订单记录等。

**扣子数据库（Coze 官方数据库）：** 提供了类似传统软件开发中数据库的功能，允许用户以表格结构存储数据。

**结构化数据表：** 创建数据库表后，默认选择单用户模式。id、sys_platform、uuid、bstudio_create_time 等字段为系统内置字段。

**导入数据：** 在测试数据中，可以点击批量导入，上传 Excel 等格式的数据文件。

**典型应用示例 - 绩效录入和查询 Agent：**
这个 Agent 支持绩效数据的查询、修改、增加、删除操作。通过意图识别节点实现功能区分：
- 用户表达查询意图时，Agent 从数据库中检索绩效数据
- 用户表达录入意图时，Agent 向数据库新增绩效记录
- 用户表达修改意图时，Agent 更新数据库中的对应记录
- 用户表达删除意图时，Agent 删除数据库中的指定记录

## 二、多模态数据处理

### 2.1 多模态数据处理概述

在使用 Coze 解决实际业务问题时，经常需要处理文本之外的多种数据类型，如图像、音频、视频等。Coze 平台通过内置的节点或插件来处理这些多模态数据，赋能各类复杂业务场景。

### 2.2 AIGC 典型应用场景

**文案生成：** 基于产品介绍和大语言模型，生成风格化、高转化率的营销文案。

**商品图片生成：** 利用文生图、图生图功能，快速生成符合商品特性的高质量展示图。

**广告语音生成：** 根据营销文案，自动生成多音色、带情感的专业广告配音。

**广告视频生成：** 结合文案、商品图片与背景音乐，自动化生成完整的短视频广告素材。

## 三、循环结构

### 3.1 什么是循环结构

循环是编程中一种基础的控制机制，核心逻辑是：重复执行一系列任务，直到满足某个预设条件为止。它能让计算机高效地处理重复性工作，避免代码冗余。

### 3.2 Coze 工作流的循环节点

在 Coze 可视化工作流中，可以直接使用内置的“循环节点”来实现循环逻辑。它专为需要批量处理或重复执行的业务场景设计，无需编写复杂的底层代码。

**典型应用场景：**
- 批量生成内容：如根据统一模板，自动为 100 个不同产品生成介绍文案
- 批量处理数据：如对海量订单数据进行自动对账、状态更新或筛选清洗
- 自动化定时任务：如每隔固定时间检查一次服务器状态，或每日生成数据报表

### 3.3 循环节点类型

**类型一：使用数组循环**
核心类比于编程语言中的 For 循环。主要用途是遍历一个已知的序列（数组），对序列中的每一个元素重复执行一系列相同的操作步骤。适用于需要处理列表中的每一项数据的场景。

**类型二：指定循环次数**
核心逻辑是在循环开始执行之前，就明确设定好具体的执行轮次。适用于任务量固定、明确知道需要重复多少次的场景，逻辑简单直接，易于控制。

**类型三：无限循环**
执行机制是循环体将持续不断地执行，直到接收到外部触发的停止信号或满足终止条件。注意事项：需极其谨慎使用，必须确保有明确的退出机制，避免程序陷入死循环导致资源耗尽。

## 四、Multi-Agent 多智能体

### 4.1 什么是 Multi-Agent

在单 Agent 模式下处理复杂任务时，需要编写非常详细和长的提示词，并且可能需要添加各种插件和工作流，这增加了调试的复杂性。调试时任何一处细节改动，都有可能影响到智能体的整体功能。实际处理用户任务时，处理结果可能与预期效果有较大出入。

为了解决上述问题，扣子提供了多 Agent 模式（Multi-Agent）。该模式下可以为智能体添加多个 Agent，并连接、配置各个 Agent 节点，通过多节点之间的分工协作来高效解决复杂的用户任务。

### 4.2 角色分工

**父 Agent（Master Agent）：** 任务的“总指挥”。负责拆解复杂任务，调度和协调各个 Agent 的工作进度。

**子 Agent（Slave Agent）：** 任务的“执行者”。职责单一明确，专注于处理父 Agent 分配的具体子任务（如搜索、计算）。

### 4.3 Multi-Agent 应用示例

构建一个类似于“小爱同学”的语音助手的智能体来控制各类设备（空调、电视、电饭煲等）。每个设备的操作和交互方式有很多种，如果把这些功能全部通过一个 Agent 来控制，那么这个 Agent 的复杂度将会非常高。

由于每个设备之间的功能都是独立互不干扰的，可以根据设备的不同拆分成多个 Agent：

**入口 Agent（父 Agent）：** 用来判断用户要控制什么设备，并调用对应的具体设备的 Agent。这个总 Agent 作为协调者，充当“总指挥”，负责接收初始任务、进行意图识别和任务分解。

**设备 Agent（子 Agent）：** 每个具体设备的 Agent 负责处理该设备下所有支持的具体操作和交互。根据设备不同，Agent 的实现也是不同的。这些子 Agent 职责单一且明确。

## 总结与学习路径

完成基础入门和进阶的学习后，应掌握以下能力：

1. **私有数据处理能力：** 能够使用 RAG 知识库和数据库功能，让智能体安全访问企业内部私有数据。

2. **多模态处理能力：** 能够利用 Coze 的多模态节点或插件，处理图像、音频、视频等非文本数据，实现 AIGC 应用。

3. **流程控制能力：** 能够熟练使用循环节点，实现批量处理和重复任务的自动化。

4. **复杂任务编排能力：** 能够设计 Multi-Agent 架构，将复杂任务拆解给多个专业 Agent 协同完成。
