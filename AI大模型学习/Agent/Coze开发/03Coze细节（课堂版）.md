# Coze 平台超详细学习笔记（完整版，含图片占位）

## 第一部分：Agent 基础与 Coze 平台入门

### 1. Agent（智能体）是什么

**定义**：Agent 是能够感知环境、分析信息、做出决策、采取行动完成任务的智能实体。

**核心公式**：

```
Agent = LLM + 记忆 + 规划 + 工具
```

- **LLM**：大语言模型作为“大脑”，负责理解和生成语言
- **记忆**：短期记忆（对话上下文）和长期记忆（向量数据库、知识库）
- **规划**：任务分解、步骤安排、决策能力
- **工具**：调用外部 API、插件、工作流等执行具体动作

![](https://raw.githubusercontent.com/1228chl/Learning-AI-Large-Models-Notes/master/Assets/Image/20260517T111350487Z.png)

---

### 2. Coze 平台概述

**Coze** 是字节跳动（ByteDance）推出的低代码智能体开发平台，以 SaaS 方式提供网页版服务。

- 国内版地址：www.coze.cn
- 国外版地址：www.coze.com
- 私有化部署版本：Coze Studio（开源版）

**对比：Coze vs Dify vs Langgraph**

| 技术栈 | 优势 | 劣势 |
|--------|------|------|
| Coze | 快速实现需求，低代码 | 限制较多，无法实现太定制化功能 |
| Dify | 快速实现需求，低代码，能对接本地模型和服务 | 本地版插件太少，很多功能需重造轮子 |
| Langgraph等框架 | 扩展性极强，基本不受限制，可对接本地模型，实现自定义复杂逻辑 | 需从零构建 |

### 3. 模型参数详解

在 Coze 中使用大模型时，有两个核心参数需要掌握：**Temperature** 和 **TopP / TopK**。

---

#### 3.1 Temperature（温度）

- **范围**：0 ~ 2
- **作用**：控制输出结果的随机性/多样性
- **高温度（如 1.5）**：输出多样性高，适合创作性场景（写诗、故事生成）
- **低温度（如 0 或 0.1）**：输出稳定性高，适合信息提取、分类、逻辑推理

**示例对比**（以电影评价词概率为例）：

| 用户问题词 | t=1.5 概率 | t=0.8 概率 | t=0.4 概率 |
|-----------|-----------|-----------|-----------|
| 震撼人心 | 0.15 | 0.25 | 0.45 |
| 引人深思 | 0.13 | 0.18 | 0.22 |
| 视觉惊艳 | 0.12 | 0.14 | 0.12 |
| 情感真挚 | 0.11 | 0.11 | 0.07 |

![](https://raw.githubusercontent.com/1228chl/Learning-AI-Large-Models-Notes/master/Assets/Image/20260517T111533655Z.png)

---

#### 3.2 TopP（核采样）

**定义**：将输出词的概率分布倒排，然后选取概率累计和大于等于 p 的最小集合。

示例：TopP=0.6 时，只保留概率最高的词直到累计概率达到 60%。

![](https://raw.githubusercontent.com/1228chl/Learning-AI-Large-Models-Notes/master/Assets/Image/20260517T111609507Z.png)

---

#### 3.3 TopK

**定义**：将输出词的概率分布倒排，只输出概率最高的前 k 个词。

示例：TopK=5，只保留概率排名前5的词。

![](https://raw.githubusercontent.com/1228chl/Learning-AI-Large-Models-Notes/master/Assets/Image/20260517T111618774Z.png)

**实用建议**：

- 需要精确回答时：Temperature=0，TopP=0.1，TopK=1
- 需要创意发散时：Temperature=0.8~1.2，TopP=0.9，TopK=50

## 第二部分：构建工作流与智能体

### 4. 工作流（Workflow）

**底层逻辑**：工作流本质是一个有向无环图（DAG），由节点和边组成。Coze 支持可视化编排。

**实操方法**：可以直接新建工作流，将已有的工作流 JSON 代码粘贴进去即可复现。

![](https://raw.githubusercontent.com/1228chl/Learning-AI-Large-Models-Notes/master/Assets/Image/20260517T111723144Z.png)

**重要提示**：

- 如果只修改工作流内部逻辑（输入输出不变），**智能体无需重新发布**，只需重新发布工作流即可生效。
- 如果修改了智能体的人设、回复逻辑或编排结构，则需要**重新发布智能体**。

## 第三部分：知识库与数据库

### 6. 知识库（Knowledge Base）

#### 6.1 为什么大模型不能完成某些任务？

三个主要原因：

1. **指令不明确** → 优化提示词工程
2. **模型不知道私有或实时数据** → 挂载知识库和数据库
3. **模型能力不足** → 换模型或微调模型

---

#### 6.2 知识库架构

**离线部分**：文档 → 切分 → Embedding（向量化） → 存入向量数据库

**在线部分**：用户问题 → Embedding（同模型） → 向量检索 → 召回相关片段 → 拼接 Prompt → LLM 生成回答

**关键点**：离线与在线必须使用**同一个 Embedding 模型**，否则语义空间不一致。

![](https://raw.githubusercontent.com/1228chl/Learning-AI-Large-Models-Notes/master/Assets/Image/20260517T111933124Z.png)

---

### 7. 数据库（Database）

#### 7.1 数据库 vs 知识库

| 维度 | 知识库 | 数据库 |
|------|--------|--------|
| 数据类型 | 非结构化数据（文档、PDF、TXT） | 结构化数据（表格、记录） |
| 存储方式 | 向量数据库（如 Milvus） | 关系型数据库（类似 MySQL） |
| 查询方式 | 语义相似度检索 | SQL 查询（精确匹配、聚合、排序） |
| 典型场景 | 知识问答、文档检索 | 绩效查询、订单管理、用户信息 |

---

#### 7.2 数据库使用注意事项

- **渠道共享**：创建数据库时必须选择“渠道共享”模式，否则可能查不出数据。
- **逻辑删除**：公司内部禁止使用物理删除（DELETE、TRUNCATE）。应使用逻辑删除，增加字段 `is_delete`，1表示删除，0表示有效。查询时加上条件 `WHERE is_delete = 0`。

## 第五部分：循环节点

### 9. 循环（Loop）

Coze 工作流中的循环节点支持三种类型：

1. **使用数组循环**：遍历数组中的每个元素执行相同操作（类似 for-each）
2. **指定循环次数**：固定次数的循环（类似 for i in range）
3. **无限循环**：需要手动设置退出条件，谨慎使用

## 第六部分：面试助手项目（完整实战）

这是一个完整的项目案例，涵盖需求分析、技术架构、工作流设计、提示词工程、API 集成等全流程。

---

### 10. 项目背景

业务痛点：公司就业服务业务模块（简历修改、面试录音分析、定制化面试题库）存在人力瓶颈，占用大量人力。需要借助大模型技术实现智能面试助手，代替部分人工。

---

### 11. 开发流程（标准软件开发流程）

1. **收集需求**：产品/领导层收集，立项报告，筛选可做项目，核心是预算
2. **需求文档**：产品编写，包含功能与指标
3. **需求评审**：产品+研发+测试+运维+各 leader，可行性分析，时间/人力投入。开发需将需求分为：可以做、不可以做、需调研
4. **技术方案**：开发输出，含技术架构、技术选型、迭代方案、风险评估、技术内部评审
5. **具体执行**：拆分任务，详细排期，与产品确认
6. **测试**：自测+测试人员。AI 开发基本自测，需满足功能要求和性能指标
7. **灰度上线/上线**：根据线上日志和指标判断，不满足则做 bad case 分析并迭代

**重要提示**：

- 项目背景必须包含**业务痛点**（人效低、指标低等）
- 通过项目解决痛点，才能产出项目成果

#### 14.3 评估与整合

**自我介绍评估提示词**（规则）：

1. 应包含姓名、学历、工作经历、项目，不可缺失
2. 工作经历突出能力，篇幅合适
3. 不要罗列太多技能点
4. 自我介绍3-5分钟，介绍最有价值的3个项目，不要一个项目说半天
5. 不要太多软素质篇幅
6. 指出违反的规则并引用原文

**问答对评估提示词**：

- 等级：A（非常好）、B（大部分对但不够深入）、C（只会一小部分或卡壳严重）、D（完全不会）
- 回答不好的需给出正确答案
- 复杂问题需先整体再细节
- 项目介绍需包含：业务背景、架构、个人工作、优化项
- 答非所问需指出理解问题

#### 15.3 项目面试题生成（关键规则）

必须包含以下问题类型：

1. **业务背景质疑**：找到可疑点提问真实性、立项牵强处 → 至少1个问题
2. **技术栈实现**：针对难点提出1-2个问题
3. **成果指标**：效率提升问分子分母；性能指标（如F1>0.9）问实现方式 → 2-3个问题；训练数据集分布
4. **优化细节**：询问实现细节及替代方案
5. **最有难度的事**：直接问“落地中最有难度的一件事，怎么克服的”
6. **技术选型**：为什么选A不选B
7. **核心技术点理解**：至少2个问题
8. **个人职责实现**：至少2个“怎么实现的”
9. **自主发挥**：根据项目特点额外提问

**输出格式**：每个问题前加项目名前缀，如“对于xxx项目：你的数据预处理部分怎么实现的？”

```
你是一个大模型算法专家，能够根据用户的项目内容提问高质量问题

注意：
1. 对于项目的业务背景部分，找到可疑的点，提问用户。尤其是是质疑背景的真实性，如果觉得项目的立项有一些牵强，可以直接问到对应的点上。至少提问一个问题
2. 对于项目的技术栈部分，结合项目个人职责的具体的内容，对于比较难实现的部分，提出1~2个问题。
3. 对于项目中所有的成果指标类的数据，比如效率提升多少，提问分子和分母是什么。对于性能类指标比如f1分数大于0.9，如果分数比较难实现，提问怎么实现的。这里提问2-3个问题，其他指标比如训练数据集多少，结合业务背景，提问用户怎么分布的
4. 如果项目中谈到了优化，针对优化项里面的内容，询问实现细节，以及是否有别的方案。
5. 直接提问：在落地这个项目中，你觉得你做的最有难度的一件事是什么，你是怎么克服的。
6. 对于项目中的技术选型问题，比如某个文本分类项目使用的bert-base-chinese作为核心技术点。询问用户为什么要使用它，而不使用其他方案

8. 针对项目中个人职责部分，问至少2个怎么实现的，之类的问题。
```

```python
    # 2 将文件和用户问题封装成消息
    message = [
        Message.build_user_question_objects(
            [
                MessageObjectString.build_file(file_id),
                MessageObjectString.build_text(user_prompt)
            ]
        )
    ]

    # 3 创建一个对话流，将消息给到agent
    stream = coze.chat.stream(
        bot_id = bot_id,
        user_id = user_id,
        additional_messages=message
    )

    # 4 循环对话流中的每一个输出，如果输出是新增，就拼接到最终结果中，如果输出是完成了或者失败了，就直接break。
    result_text = ""
    # 对话流中拿出具体的信息
    for s in stream:
        # print(s)
        if s.event == ChatEventType.CONVERSATION_MESSAGE_DELTA:
            result_text += s.message.content
        elif s.event in [ChatEventType.CONVERSATION_CHAT_FAILED,
                                 ChatEventType.CONVERSATION_CHAT_COMPLETED]:
            break


    # 5 存储agent输出的最终结果。
    print(result_text)
    with open(RESULT_PATH, "w", encoding="utf-8") as f:
        f.write(result_text)

    return result_text
```

## 第七部分：话术文档（面试准备）

简历是面试的“考题”，话术文档就是“答案”。初版话术需一直保留，每次面试后更新补充。

### 话术文档包含四部分：

#### 1. 个人介绍（约3分钟）

- 基本信息（姓名、籍贯等）
- 毕业学校/学历（好则说，不好则不说）
- 专业技能（擅长哪些）
- 工作经历（公司、岗位、负责什么）
- 项目经历（做过几个项目，分别是什么，一两句话说明）
- 最近项目是xxx
- 感谢面试机会

---

#### 2. 项目介绍（5-10分钟）

- 项目背景（痛点：人效低、准确率低）
- 项目框架（或技术栈）
- 项目步骤/流程（分成三大步，每步3-5句话，体现技术栈）
- 项目效果（人效提升/准确率提升多少）
- 额外准备：项目中遇到的2-3个问题及解决方案，项目优化点

---

#### 3. 面试问题库

记录过往面试中遇到的所有问题和自己的回答，持续复盘。

---

#### 4. 公司情况调研

- 主营业务
- 团队构成：组长（技术leader）、开发、高级开发（1-2个）
- 对接人：产品经理、测试、运维、领导、后端、前端
