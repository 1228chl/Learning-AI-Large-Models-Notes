## 一、详细笔记

### 第1部分：多任务的介绍

#### 1.1 多任务的概念

- **定义**：多任务是指在同一时间内执行多个任务（给人的感觉）。实际上，对于单核CPU，是交替执行；对于多核CPU，可能是真正并行。
- **生活实例**：现在的电脑操作系统都是多任务操作系统，可以同时运行多个软件（如一边听歌、一边写文档、一边上网）。

#### 1.2 多任务的作用

- **充分利用CPU资源**：如果程序是单任务的，一个函数执行完才能执行另一个，CPU会有空闲等待时间。多任务可以让CPU始终忙碌。
- **大大提高程序执行效率**：例如下载文件时，多个文件同时下载比一个一个下载快得多（因为网络IO等待时间可以被重叠）。

#### 1.3 多任务的两种表现形式

| 类型         | 并发 (Concurrency)                                           | 并行 (Parallelism)                                      |
| ------------ | ------------------------------------------------------------ | ------------------------------------------------------- |
| **定义**     | 在一段时间内**交替**执行多个任务                             | 在一段时间内**真正的同时一起**执行多个任务              |
| **硬件基础** | 单核CPU                                                      | 多核CPU                                                 |
| **原理**     | 操作系统轮流让各个任务交替执行（例如每个任务执行0.01秒），由于CPU速度极快，感觉像同时执行 | 操作系统给每个CPU内核安排一个任务，多个内核真正同时执行 |
| **图示理解** | 任务A→任务B→任务A→任务B…（快速切换）                         | 任务A和任务B同时运行                                    |

> **注意**：并发是逻辑上的同时，并行是物理上的同时。现代操作系统通常混合使用并发和并行。

#### 1.4 思考题

- **问题**：之前所写的程序都是单任务的（一个函数执行完才能执行另一个），能否让多个任务同时执行？
- **答案**：不能。需要借助多任务编程技术，如多进程、多线程。

### 第2部分：进程的介绍

#### 2.1 进程的概念

- **进程（Process）**：是**CPU资源分配的最小单位**，也是操作系统进行资源分配和调度运行的基本单位。
- **通俗理解**：一个正在运行的程序就是一个进程。例如，正在运行的QQ、微信、浏览器等，每个都是一个独立的进程。
- **注意**：一个程序运行后至少有一个进程（即主进程）。

#### 2.2 多进程的作用

- **实现多任务**：多进程是Python中实现多任务的一种重要方式。
- **提高效率**：通过创建多个进程，可以让多个任务“同时”执行，充分利用多核CPU，大大提升程序执行效率。

#### 2.3 多进程基本工作方式

```
程序运行（如 hello.py）
    │
    ├── 默认创建主进程
    │       └── 执行 func_a
    │
    └── 可以在主进程中创建子进程
            └── 子进程执行 func_b
            （主进程和子进程并发/并行执行）
```

- 程序运行起来，操作系统会为它创建一个进程，称为**主进程**。
- 在主进程中，可以通过代码创建新的进程，称为**子进程**。
- 主进程和子进程各自执行不同的任务，从而实现多任务。

### 第3部分：多进程完成多任务

#### 3.1 进程创建步骤

1. **导入进程工具包**

   ```python
   import multiprocessing
   ```

2. **通过进程类实例化进程对象**

   ```python
   子进程对象 = multiprocessing.Process(target=任务名)
   ```

3. **启动进程执行任务**

   ```python
   子进程对象.start()
   ```

#### 3.2 Process类参数详解

```python
multiprocessing.Process(group=None, target=None, name=None, args=(), kwargs={})
```

| 参数   | 说明                                                         |
| ------ | ------------------------------------------------------------ |
| group  | 参数未使用，值始终为None                                     |
| target | 表示调用对象，即子进程要执行的任务（函数名，不带括号）       |
| args   | 以**元组**形式向任务函数传参，元组中元素的顺序必须与任务函数参数的顺序一致 |
| kwargs | 以**字典**形式向任务函数传参，字典中的key必须与任务函数的参数名一致 |
| name   | 子进程的名称（可选）                                         |

#### 3.3 无参进程示例

```python
import multiprocessing
import time

def coding():
    for i in range(3):
        print("coding...")
        time.sleep(0.2)

def music():
    for i in range(3):
        print("music...")
        time.sleep(0.2)

if __name__ == '__main__':
    coding_process = multiprocessing.Process(target=coding)
    music_process = multiprocessing.Process(target=music)
    coding_process.start()
    music_process.start()
```

**输出**（可能交替出现）：

```
coding...
music...
coding...
music...
coding...
music...
```

#### 3.4 有参进程示例

**元组传参**：

```python
def coding(num, name):
    for i in range(num):
        print(f"{name}正在编写第{i}行代码...")
        time.sleep(0.2)

coding_process = multiprocessing.Process(target=coding, args=(3, '小明'))
```

**字典传参**：

```python
def music(count, name):
    for i in range(count):
        print(f"{name}正在听第{i}首歌...")
        time.sleep(0.2)

music_process = multiprocessing.Process(target=music, kwargs={'count': 2, 'name': '小明'})
```

> **注意**：
> - 元组传参时，参数的顺序必须与函数定义顺序完全一致。
> - 字典传参时，字典的key必须与函数参数名一致。

### 第4部分：获取进程编号

#### 4.1 进程编号的作用

- **唯一标识**：每个进程在操作系统中都有一个唯一的编号（PID），用于区分不同的进程。
- **管理进程**：方便操作系统和用户对进程进行管理（如终止、查看状态等）。
- **验证关系**：通过获取当前进程的PID和父进程的PPID，可以验证主进程和子进程的关系，知道子进程是由哪个主进程创建的。

> **注意**：进程号可以反复使用。当一个进程终止后，它的PID可以被新创建的进程使用。

#### 4.2 获取当前进程编号

**方法1：使用os模块**

```python
import os
pid = os.getpid()
print("当前进程编号:", pid)
```

**方法2：使用multiprocessing模块**

```python
import multiprocessing
pid = multiprocessing.current_process().pid
print("当前进程编号:", pid)
```

#### 4.3 获取父进程编号

```python
import os
ppid = os.getppid()
print("父进程编号:", ppid)
```

#### 4.4 示例：验证主进程和子进程的关系

```python
import multiprocessing
import os

def sub_task():
    print("子进程PID:", os.getpid())
    print("子进程的父进程PID:", os.getppid())

if __name__ == '__main__':
    print("主进程PID:", os.getpid())
    p = multiprocessing.Process(target=sub_task)
    p.start()
    p.join()
```

输出类似：

```
主进程PID: 12345
子进程PID: 12346
子进程的父进程PID: 12345
```

### 第5部分：进程的注意点

#### 5.1 注意点1：进程之间不共享全局变量

- **现象**：在一个进程中修改全局变量，另一个进程读取不到修改后的值。

**验证代码**：

```python
import multiprocessing
import time

my_list = []   # 全局变量

def write_data():
    for i in range(3):
        my_list.append(i)
        print("add:", i)
    print("write_data:", my_list)

def read_data():
    print("read_data:", my_list)

if __name__ == '__main__':
    write_process = multiprocessing.Process(target=write_data)
    read_process = multiprocessing.Process(target=read_data)
    write_process.start()
    time.sleep(1)   # 确保写入进程先执行
    read_process.start()
```

**输出**：

```
add: 0
add: 1
add: 2
write_data: [0, 1, 2]
read_data: []      # 读取进程看到的仍然是空列表
```

- **原因**：创建子进程时，会对主进程的资源进行**拷贝**。子进程是主进程的一个副本，相当于一对双胞胎。不同进程操作的不是同一个全局变量，只是变量名相同而已。每个进程都有自己独立的内存空间，互不影响。

> **对比线程**：线程之间共享全局变量，进程之间不共享。这是进程和线程的重要区别之一。

#### 5.2 注意点2：主进程会等待所有子进程执行结束再结束

- **现象**：主进程的代码执行完毕后，程序不会立即退出，而是等待所有子进程执行完成后才退出。

**验证代码**：

```python
import multiprocessing
import time

def work():
    for i in range(10):
        print("工作中...")
        time.sleep(0.2)

if __name__ == '__main__':
    work_process = multiprocessing.Process(target=work)
    work_process.start()
    time.sleep(1)   # 主进程只执行1秒
    print("主进程执行完毕")
```

**输出**：

```
工作中...
工作中...
...（继续输出，直到子进程执行完10次）
主进程执行完毕
```

可以看到，“主进程执行完毕”打印后，程序并没有立即退出，而是等子进程全部执行完才退出。

- **原因**：为了保证子进程能够正常运行，主进程会等待所有子进程执行完成以后再销毁。

#### 5.3 不让主进程等待子进程的方法

如果希望主进程退出时，子进程立即销毁不再执行，有两种方法：

##### 方法1：设置守护主进程（推荐）

```python
if __name__ == '__main__':
    work_process = multiprocessing.Process(target=work)
    work_process.daemon = True   # 设置为守护进程
    work_process.start()
    time.sleep(1)
    print("主进程执行完毕")
    # 主进程退出时，子进程会被自动销毁
```

**效果**：主进程退出后，子进程立即终止，不再继续执行。

##### 方法2：手动终止子进程（不推荐）

```python
if __name__ == '__main__':
    work_process = multiprocessing.Process(target=work)
    work_process.start()
    time.sleep(1)
    work_process.terminate()   # 强制终止子进程
    print("主进程执行完毕")
```

> **警告**：`terminate()`方法会立即终止子进程，但可能产生**僵尸进程**，且不会清理资源。一般不推荐使用，优先使用守护进程方式。

#### 5.4 进程注意点总结

| 注意点           | 说明                                           |
| ---------------- | ---------------------------------------------- |
| 全局变量不共享   | 每个进程有独立的内存空间，子进程拷贝主进程资源 |
| 主进程等待子进程 | 默认行为，确保子进程正常执行                   |
| 守护进程         | 设置`daemon=True`，主进程退出时子进程自动销毁  |
| 手动终止         | `terminate()`方法，不推荐（可能产生僵尸进程）  |

## 二、进程与线程对比速查表

| 对比项  | 进程                  | 线程                  |
| ---- | ------------------- | ------------------- |
| 资源分配 | 操作系统资源分配的基本单位       | 依附于进程，不独立分配资源       |
| 调度   | CPU调度的基本单位（线程是调度单位） | CPU调度的基本单位          |
| 全局变量 | 不共享（独立内存空间）         | 共享（需注意资源竞争）         |
| 创建开销 | 大（需要分配独立内存）         | 小（共享进程内存）           |
| 多核利用 | 可以充分利用多核（真正并行）      | 受GIL限制，不能并行计算密集型任务  |
| 稳定性  | 强（一个进程崩溃不影响其他进程）    | 弱（一个线程崩溃可能导致整个进程崩溃） |
| 通信方式 | IPC（管道、队列、共享内存等）复杂  | 直接读写共享变量（需加锁）       |
| 适用场景 | 计算密集型、需要高稳定性        | I/O密集型、需要低开销并发      |

## 四、常见面试/考试题

1. **什么是多任务？多任务的两种表现形式是什么？**
   - 多任务：同一时间内执行多个任务。
   - 并发：交替执行；并行：同时执行。

2. **进程和程序的区别？**
   - 程序是静态的代码文件，进程是程序的一次动态执行。

3. **如何创建多进程？**
   - 导入`multiprocessing`，创建`Process`对象，调用`start()`。

4. **进程之间为什么不能共享全局变量？**
   - 因为子进程会拷贝主进程的资源，每个进程有独立的内存空间。

5. **主进程会等待子进程结束吗？如何改变？**
   - 默认会等待。设置`daemon=True`可以让主进程退出时子进程销毁。

6. **获取当前进程编号和父进程编号的方法？**
   - `os.getpid()`，`os.getppid()`。
