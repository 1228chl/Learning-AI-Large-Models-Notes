# Python 闭包和装饰器 

## 一、详细笔记

### 第1部分：闭包

#### 1.1 闭包的作用

- **问题背景**：在之前的学习中，当函数调用完成后，函数内部定义的变量就会被销毁（释放内存）。但有时我们需要**保存函数内的变量**，并在这个变量的基础上完成一系列操作（例如每次在这个变量的基础上累加数值）。
- **闭包的作用**：闭包可以**保存函数内的变量**，而不会随着函数调用结束而被销毁。闭包实现了类似“持久化”局部变量的效果。

---

#### 1.2 闭包的语法格式

```python
# 外部函数
def 外部函数名(外部参数):
    # 内部函数
    def 内部函数名(内部参数):
        # 使用了外部函数的变量（包括参数）
        ...
    return 内部函数名      # 返回内部函数（注意：不要加括号，返回的是函数对象）
```

**调用方式**：

```python
闭包实例 = 外部函数名(外部参数值)   # 得到内部函数对象
闭包实例(内部参数值)              # 执行内部函数
```

---

#### 1.3 闭包的构成条件（三个必要条件）

| 条件       | 说明                                               |
| ---------- | -------------------------------------------------- |
| **有嵌套** | 在函数内部再定义函数（函数嵌套）                   |
| **有引用** | 内部函数使用了外部函数的变量（包括外部函数的参数） |
| **有返回** | 外部函数返回了内部函数名（即返回函数对象）         |

> 缺少任何一个条件，都不构成闭包。

---

#### 1.4 闭包示例：求和

```python
# 外部函数
def func_out(num1):
    # 内部函数
    def func_inner(num2):
        num = num1 + num2   # 内部函数使用了外部函数的参数 num1
        print("现在的值：", num)
    # 外部函数返回内部函数
    return func_inner

# 创建闭包实例
f = func_out(10)   # 此时 f 就是 func_inner，并且记住了 num1=10
# 执行闭包
f(1)   # 输出：现在的值：11
f(2)   # 输出：现在的值：12
```

**执行流程分析**：

1. 调用 `func_out(10)`，执行外部函数，定义内部函数 `func_inner`，并返回内部函数对象（尚未执行内部函数）。
2. `f = func_out(10)`，`f` 指向内部函数，并且内部函数“记住”了外部函数中的 `num1=10`（形成闭包）。
3. 调用 `f(1)` 时，执行内部函数，使用已保存的 `num1=10` 和传入的 `num2=1`，计算并打印。

---

#### 1.5 nonlocal 关键字

- **问题**：在内部函数中，默认只能**读取**外部函数的变量，但不能直接**修改**。如果尝试修改，Python会认为你在内部函数中创建了一个新的局部变量，而不是修改外部函数的变量。
- **解决方案**：使用 `nonlocal` 关键字声明变量，表示该变量来自**外层函数（嵌套作用域）**，允许修改。
- **对比**：
  - `global`：声明全局变量
  - `nonlocal`：声明外层函数（非全局）的变量

**示例：使用 nonlocal 修改外部函数变量**

```python
def func_out():
    a = 100
    def func_inner():
        nonlocal a     # 声明 a 来自外部函数
        a = a + 1      # 修改外部函数的 a
        print(f"修改的结果: {a}")
    return func_inner

number = func_out()
number()   # 输出：修改的结果: 101
number()   # 输出：修改的结果: 102（a的值被保留）
```

> **注意**：如果不加 `nonlocal`，直接 `a = a + 1` 会报错（UnboundLocalError），因为 `a += 1` 中的 `a` 被认为是局部变量，但赋值前没有定义。

---

#### 1.6 闭包总结

| 问题                    | 答案                                           |
| ----------------------- | ---------------------------------------------- |
| 闭包的作用？            | 保存函数中的变量（外部函数中的变量），不被销毁 |
| 构成闭包的三个条件？    | 有嵌套、有引用、有返回                         |
| nonlocal 关键字的作用？ | 声明让内部函数能够修改外部函数的变量           |

### 第2部分：装饰器

#### 2.1 装饰器概述

- **装饰器的作用**：在**不改变原有函数的基础上**，给原有函数增加额外的功能。例如：日志记录、权限验证、性能测试、缓存等。
- **装饰器的本质**：装饰器就是一个**闭包函数**。
- **装饰器的构成条件**（在闭包三个条件基础上增加一个）：

| 条件       | 说明                                                     |
| ---------- | -------------------------------------------------------- |
| 有嵌套     | 函数内部定义函数                                         |
| 有引用     | 内部函数使用了外部函数的变量（通常是被装饰的函数名）     |
| 有返回     | 外部函数返回内部函数                                     |
| 有额外功能 | 内部函数中添加需要增加的新功能（如打印日志、权限校验等） |

---

#### 2.2 装饰器的使用方式

##### 方式1：传统方式（手动装饰）

```python
# 定义装饰器（闭包）
def check(fn):
    def inner():
        print("登陆验证...")   # 额外功能
        fn()                   # 调用原函数
    return inner

# 原函数
def comment():
    print("发表评论")

# 手动装饰
comment = check(comment)   # 将原函数替换为装饰后的函数
comment()                  # 调用装饰后的函数
```

##### 方式2：语法糖方式（推荐，最常见）

```python
def check(fn):
    def inner():
        print("请先登陆")
        fn()
    return inner

@check        # 等价于 comment = check(comment)
def comment():
    print("发表评论")

comment()     # 调用装饰后的函数
```

> **注意**：`@装饰器名` 必须紧贴在被装饰函数的定义上方。解释器遇到 `@check` 时会立即执行 `comment = check(comment)`。

---

#### 2.3 装饰器装饰不同类型函数

原函数可能有参数、可能有返回值，装饰器中的内部函数需要与原函数**保持一致的签名**（参数和返回值）。

---

##### 2.3.1 装饰无参无返回值的函数

```python
def print_info(func):
    def inner():
        print("【友好提示】正在努力计算中...")
        func()          # 调用原函数，无返回值
    return inner

@print_info
def get_sum():
    a, b = 10, 20
    c = a + b
    print(f"两数之和为: {c}")

get_sum()
```

---

##### 2.3.2 装饰有参无返回值的函数

```python
def print_info(func):
    def inner(a, b):          # 内部函数接收参数
        print("【友好提示】正在努力计算中...")
        func(a, b)            # 传递参数给原函数
    return inner

@print_info
def get_sum(x, y):
    z = x + y
    print(f"两数之和为: {z}")

get_sum(20, 35)
```

---

##### 2.3.3 装饰无参有返回值的函数

```python
def print_info(func):
    def inner():
        print("【友好提示】正在努力计算中...")
        result = func()       # 接收原函数的返回值
        return result         # 返回该返回值
    return inner

@print_info
def get_sum():
    x, y = 19, 34
    return x + y

print(get_sum())   # 输出计算结果
```

---

##### 2.3.4 装饰有参有返回值的函数

```python
def print_info(func):
    def inner(a, b):
        print("【友好提示】正在努力计算中...")
        result = func(a, b)
        return result
    return inner

@print_info
def get_sum(x, y):
    return x + y

print(get_sum(13, 45))
```

---

##### 2.3.5 通用装饰器（不定长参数，有返回值）

对于参数个数不确定的函数，可以使用 `*args` 和 `**kwargs` 来实现通用装饰器。

```python
def print_info(func):
    def inner(*args, **kwargs):
        print("【友好提示】正在努力计算中...")
        return func(*args, **kwargs)   # 将接收到的所有参数原样传递给原函数
    return inner

@print_info
def get_sum(*args, **kwargs):
    result = 0
    for value in args:
        result += value
    for value in kwargs.values():
        result += value
    return result

# 调用：可以传递任意数量的位置参数和关键字参数
print(get_sum(1, 3, 4, 5, a=10, b=20, c=30))
```

**总结规则**：

- 原函数有参数，装饰器的内部函数也要有参数（数量/类型匹配，或用 `*args, **kwargs`）。
- 原函数有返回值，装饰器的内部函数也要有返回值（接收并返回）。

---

#### 2.4 多个装饰器的使用

一个函数可以被多个装饰器装饰。语法如下：

```python
@decorator1
@decorator2
def func():
    pass
```

**装饰顺序（执行顺序）**：**由内到外**（离函数最近的装饰器先装饰，然后外面的装饰器再装饰）。

- `@decorator2` 靠近函数，先执行 `func = decorator2(func)`
- 然后 `@decorator1` 执行 `func = decorator1(func)`
- 最终调用时，先执行 `decorator1` 的额外功能，再执行 `decorator2` 的额外功能，最后执行原函数。

**案例：发表评论前先验证用户登录，再验证验证码**

```python
# 装饰器1：检查用户登录
def check_user(fn):
    def inner():
        print("输入用户名和密码")   # 模拟登录验证
        fn()
    return inner

# 装饰器2：检查验证码
def check_code(fn):
    def inner():
        print("登录验证码")         # 模拟验证码验证
        fn()
    return inner

@check_user      # 外层装饰器（后执行装饰，先执行其内部额外功能）
@check_code      # 内层装饰器（先执行装饰）
def comment():
    print("发表评论")

comment()
```

**输出**：

```
输入用户名和密码
登录验证码
发表评论
```

**解释**：

1. `@check_code` 先装饰，`comment = check_code(comment)`，此时 `comment` 指向 `check_code` 的 `inner`。
2. `@check_user` 再装饰，`comment = check_user(已经装饰过的comment)`，最终 `comment` 指向 `check_user` 的 `inner`。
3. 调用 `comment()` 时，先执行 `check_user.inner` 中的打印“输入用户名和密码”，然后调用被装饰的函数（即 `check_code.inner`），执行“登录验证码”，再调用原 `comment` 打印“发表评论”。

#### 2.5 带有参数的装饰器

##### 场景

同一个装饰器可能根据不同的配置表现出不同的行为（例如：加法运算显示“加法提示”，减法运算显示“减法提示”）。这就需要装饰器能够接收参数。

##### 实现方式

**方式一：原始方式（不使用语法糖）**

装饰器函数直接接收额外参数，但这种方式在语法糖下无法直接使用（因为 `@decorator` 后面只能跟装饰器名，不能带参数）。

```python
def decorator(fn, flag):
    def inner(num1, num2):
        if flag == "+":
            print("--正在努力加法计算--")
        elif flag == "-":
            print("--正在努力减法计算--")
        result = fn(num1, num2)
        return result
    return inner

def add(a, b):
    return a + b

# 手动装饰时传入参数
add = decorator(add, "+")
print(add(1, 3))
```

**方式二：语法糖方式（三层嵌套）**

为了支持 `@装饰器名(参数)` 的语法，需要在装饰器外面再包裹一层函数，该函数接收参数，返回真正的装饰器。

```python
# 最外层：接收参数的函数
def logging(flag):
    # 中间层：真正的装饰器（接收原函数）
    def decorator(fn):
        # 内层：内部函数（接收原函数的参数）
        def inner(num1, num2):
            if flag == "+":
                print("--正在努力加法计算--")
            elif flag == "-":
                print("--正在努力减法计算--")
            result = fn(num1, num2)
            return result
        return inner
    return decorator

# 使用带有参数的装饰器
@logging("+")   # 等价于 add = logging("+")(add)
def add(a, b):
    return a + b

@logging("-")
def reduce(a, b):
    return a - b

print(add(1, 3))      # 输出加法提示并计算结果 4
print(reduce(5, 2))   # 输出减法提示并计算结果 3
```

**执行流程解析**：

- `@logging("+")` 首先调用 `logging("+")`，返回 `decorator` 函数。
- 然后 `@decorator` 装饰 `add`，即 `add = decorator(add)`。
- `decorator(add)` 返回 `inner` 函数，所以 `add` 最终指向 `inner`。

**三层结构总结**：

- **最外层函数**：接收装饰器的配置参数，返回一个装饰器。
- **中间层函数**：真正的装饰器，接收被装饰的函数，返回内部函数。
- **内层函数**：内部函数，接收原函数的参数，添加额外功能，调用原函数并返回结果。

## 四、常见面试/考试题

1. **什么是闭包？闭包需要满足哪些条件？**
   - 闭包是函数嵌套中，内部函数使用了外部函数的变量，并且外部函数返回了内部函数。条件：有嵌套、有引用、有返回。

2. **`nonlocal` 和 `global` 的区别？**
   - `global` 声明全局变量；`nonlocal` 声明外层函数（嵌套作用域）中的变量，用于在内部函数中修改它。

3. **装饰器的作用是什么？**
   - 在不改变原函数代码和调用方式的前提下，动态地为函数添加额外功能。

4. **装饰器的本质是什么？**
   - 闭包函数。

5. **如何实现一个装饰器，使其能够装饰任何参数和任何返回值的函数？**
   - 使用 `*args, **kwargs` 接收任意参数，并在内部调用 `func(*args, **kwargs)` 并返回其结果。

6. **多个装饰器装饰一个函数时，装饰顺序和执行顺序是怎样的？**
   - 装饰顺序：由内到外（靠近函数的先装饰）。执行顺序：调用时由外到内执行各装饰器的额外功能，最后执行原函数。

7. **如何实现一个带参数的装饰器？**
   - 在标准装饰器外面再包裹一层函数，该函数接收参数，返回一个装饰器。最终形成三层嵌套结构。
