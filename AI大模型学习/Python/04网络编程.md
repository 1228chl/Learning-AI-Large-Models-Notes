# Python 网络编程

---

## 一、详细笔记

### 第1部分：网络编程三要素

#### 1.1 基本概念

- **网络**：将具有独立功能的多台计算机通过通信线路和通信设备连接起来，在网络管理软件及网络通信协议下，实现资源共享和信息传递的虚拟平台。
- **网络编程**：编写基于网络通信的软件或程序，实现多台计算机之间的数据交换。
- **网络编程三要素**：IP地址、端口号、协议。  
  - 通过IP地址找到指定计算机；  
  - 通过端口号找到计算机上指定的进程（应用程序）；  
  - 通过协议确保通信双方按规则交换数据。

> **生活类比**：要与小明对话：  
> 1. 找到小明的住处（IP地址）；  
> 2. 小明用耳朵听（端口）；  
> 3. 双方都说普通话（协议）。

---

#### 1.2 IP地址

- **概念**：IP地址是标识网络中设备的一个地址，好比现实生活中的家庭地址。
- **分类**：
  - **IPv4**：当前广泛使用，32位地址，如`192.168.1.1`。
  - **IPv6**：未来使用，128位地址，解决IPv4地址枯竭问题。
- **作用**：通过网络唯一标识一台设备，找到它并进行数据通信。

- **常用命令**：
  - 查看IP地址：
    - Linux / macOS：`ifconfig`
    - Windows：`ipconfig`
  - 检查网络连通性（`ping`命令）：
    - `ping www.baidu.com` — 检查是否能上公网
    - `ping 192.168.1.xxx` — 检查是否在同一局域网
    - `ping 127.0.0.1` — 检查本机网卡是否正常（回环地址）

---

#### 1.3 端口和端口号

- **端口**：传输数据的通道，好比教室的门，是数据传输必经之路。  
  **注意**：只有已经运行的程序才会被分配端口号；没有运行的程序不分配端口号。

- **端口号**：标识电脑中唯一端口的数字，取值范围 0 ~ 65535。  
  通过端口号可以找到对应的端口，进而将数据交给对应的应用程序。

- **端口号分类**：
  - **知名端口号**（0 ~ 1023）：众所周知的端口号，固定分配给特定服务。  
    例如：HTTP（80）、HTTPS（443）、FTP（21）、SSH（22）。
  - **动态端口号**（1024 ~ 65535）：一般程序员开发应用程序时使用，可以自定义。

---

#### 1.4 协议（TCP）

- **TCP概念**：TCP（Transmission Control Protocol，传输控制协议）是一种**面向连接的、可靠的、基于字节流的传输层通信协议**。
- **TCP特点**：
  - **面向连接**：通信双方必须先建立连接才能传输数据，通信结束后必须断开连接。
  - **可靠传输**：通过确认、重传、排序等机制保证数据完整、有序到达。
  - 类似于生活中的**打电话**：拨通→通话→挂断。

- **TCP通信步骤**：  
  1. 创建连接（三次握手）  
  2. 传输数据  
  3. 关闭连接（四次挥手）

---

##### 1.4.1 三次握手（建立连接）

三次握手是指建立一个TCP连接时，客户端和服务端总共发送3个包以确认连接的建立。

| 步骤       | 方向            | 内容                 | 说明                                             |
| ---------- | --------------- | -------------------- | ------------------------------------------------ |
| 第一次握手 | 客户端 → 服务端 | 发送请求（随机数x）  | 客户端请求建立连接，等待服务端确认               |
| 第二次握手 | 服务端 → 客户端 | 回复（x+1，随机数y） | 服务端收到请求，确认连接请求，并发出自己的同步包 |
| 第三次握手 | 客户端 → 服务端 | 回复（y+1）          | 客户端确认服务端的同步包，连接建立成功           |

完成三次握手后，客户端与服务端之间可以开始传输数据。

> **简单记忆**：  
> A：“我要连接你”（seq=x）  
> B：“好的，我收到了，我也要连接你”（ack=x+1, seq=y）  
> A：“好的，收到你的连接”（ack=y+1）

---

##### 1.4.2 四次挥手（断开连接）

四次挥手是TCP断开连接时需要经过4次确认。由于TCP连接是双向的（A→B 和 B→A），每一方都需要单独关闭自己的传输方向。

| 步骤       | 方向          | 内容                     | 说明                                 |
| ---------- | ------------- | ------------------------ | ------------------------------------ |
| 第一次挥手 | 主机A → 主机B | 发送请求（随机数u）      | A提出停止A→B方向的连接请求           |
| 第二次挥手 | 主机B → 主机A | 回复（u+1）              | B确认收到，关闭A→B方向连接           |
| 第三次挥手 | 主机B → 主机A | 发送消息（u+1，随机数w） | B提出停止B→A方向的连接请求           |
| 第四次挥手 | 主机A → 主机B | 回复（w+1）              | A确认，关闭B→A方向连接，连接彻底断开 |

> **注意**：四次挥手完成后，双方关闭连接，释放资源。

### 第2部分：Socket套接字与TCP开发流程

#### 2.1 Socket套接字

- **概念**：Socket（套接字）是进程之间通信的工具，好比生活中的**插座**，所有电器都需要插在插座上才能工作。进程之间想要进行网络通信需要基于Socket。
- **作用**：负责进程之间的网络数据传输，是数据的搬运工。  
  几乎所有与网络相关的应用程序都使用了Socket。
- **Python中使用Socket**：

  ```python
  import socket
  tcp_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
  ```

  - `AF_INET`：使用IPv4地址族
  - `SOCK_STREAM`：使用TCP协议（流式套接字）

---

#### 2.2 TCP开发流程概述

TCP网络应用程序开发分为两部分：

- **TCP客户端程序**：运行在用户设备上的程序（如浏览器）。
- **TCP服务端程序**：运行在服务器设备上的程序，专门为客户端提供数据服务（如Web服务器）。

---

#### 2.3 TCP服务器端开发流程（详细步骤）

| 步骤                  | 方法/操作          | 说明                                                   |
| --------------------- | ------------------ | ------------------------------------------------------ |
| 1. 创建服务端套接字   | `socket.socket()`  | 创建被动套接字（只负责接收新连接）                     |
| 2. 绑定端口号         | `bind((ip, port))` | 将套接字绑定到本机IP和端口，IP为空字符串表示本机所有IP |
| 3. 设置监听           | `listen(backlog)`  | backlog表示最大等待连接数（如128）                     |
| 4. 等待接受客户端连接 | `accept()`         | 阻塞等待，返回(新套接字, 客户端地址)                   |
| 5. 接收数据           | `recv(bufsize)`    | 从客户端接收数据（二进制），bufsize为最大字节数        |
| 6. 发送数据           | `send(data)`       | 向客户端发送数据（需编码为bytes）                      |
| 7. 关闭套接字         | `close()`          | 先关闭与客户端通信的套接字，再关闭被动套接字           |

> **注意**：
> - `accept()` 之前的套接字（被动套接字）**不能**用于收发消息，只负责接收新连接。
> - `accept()` 返回的**新套接字**专门用于与当前客户端通信。

---

#### 2.4 TCP客户端开发流程（详细步骤）

| 步骤                | 方法/操作             | 说明                         |
| ------------------- | --------------------- | ---------------------------- |
| 1. 创建客户端套接字 | `socket.socket()`     | 创建主动套接字               |
| 2. 连接服务器       | `connect((ip, port))` | 连接到指定服务器的IP和端口   |
| 3. 发送数据         | `send(data)`          | 向服务器发送请求数据（编码） |
| 4. 接收数据         | `recv(bufsize)`       | 接收服务器响应数据（解码）   |
| 5. 关闭套接字       | `close()`             | 关闭连接                     |

---

#### 2.5 字符串与二进制转换（编码/解码）

在网络中，数据是以**二进制类型（bytes）**的形式进行传递的。

- **编码（str → bytes）**：将字符串转换为二进制数据，使用 `encode()` 方法。

  ```python
  strs = "黑马程序员"
  bs = strs.encode(encoding="utf-8")
  print(bs)   # b'\xe9\xbb\x91\xe9\xa9\xac...'
  ```

- **解码（bytes → str）**：将二进制数据转换为字符串，使用 `decode()` 方法。

  ```python
  bs = b"itheima"
  result = bs.decode(encoding="utf-8")
  print(result)   # itheima
  ```

> 注意：编码和解码必须使用相同的字符集（如 `utf-8`）。

### 第3部分：TCP编程（完整代码实现与注意点）

#### 3.1 TCP服务器端完整代码

```python
import socket

if __name__ == '__main__':
    # 1. 创建服务端套接字对象（TCP）
    tcp_server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

    # 2. 绑定IP地址和端口号
    # 如果IP地址设置为""，表示绑定本机所有可用的IP地址
    tcp_server_socket.bind(("", 8888))   # 绑定8888端口

    # 3. 设置监听，128表示最大等待连接数
    tcp_server_socket.listen(128)

    # 4. 等待接受客户端的连接请求（阻塞）
    print("等待客户端连接...")
    conn_socket, ip_port = tcp_server_socket.accept()
    print("客户端地址:", ip_port)

    # 5. 接收客户端发送的数据
    recv_data = conn_socket.recv(1024)   # 最多接收1024字节
    print("接收到的数据:", recv_data.decode())

    # 6. 发送数据给客户端
    conn_socket.send("客户端你的数据我收到了".encode())

    # 7. 关闭套接字
    conn_socket.close()          # 关闭与当前客户端的连接
    tcp_server_socket.close()    # 关闭服务端被动套接字
```

---

#### 3.2 TCP客户端完整代码

```python
import socket

if __name__ == '__main__':
    # 1. 创建客户端套接字对象（TCP）
    tcp_client_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

    # 2. 连接服务端
    tcp_client_socket.connect(("127.0.0.1", 8888))   # 连接本机8888端口

    # 3. 发送数据
    tcp_client_socket.send("你好，服务器".encode("utf-8"))

    # 4. 接收数据（阻塞）
    recv_data = tcp_client_socket.recv(1024)
    print("收到服务器响应:", recv_data.decode())

    # 5. 关闭套接字
    tcp_client_socket.close()
```

---

#### 3.3 TCP网络程序注意点（重要）

| 注意点                       | 详细说明                                                     |
| ---------------------------- | ------------------------------------------------------------ |
| **服务端必须绑定端口**       | 否则客户端无法找到服务端程序。建议同时绑定IP（`""`表示本机所有IP） |
| **被动套接字 vs 通信套接字** | `accept()` 前的套接字只能接收连接，不能收发消息；`accept()` 返回的新套接字用于与特定客户端通信 |
| **关闭连接套接字**           | 关闭 `conn_socket` 表示与该客户端通信完毕；关闭 `tcp_server_socket` 表示服务端不再接收新连接 |
| **服务端关闭慎重**           | 服务端套接字关闭后，所有客户端无法再连接，通常服务端不主动关闭 |
| **客户端下线检测**           | 当客户端调用 `close()` 后，服务端的 `recv()` 会立即解除阻塞，返回的数据长度为0，可用于判断客户端已断开 |
| **端口占用问题**             | 服务端程序退出后，端口不会立即释放，需要等待约1~2分钟。解决方法：**设置端口复用** |

---

#### 3.4 端口复用设置

在服务端 `bind()` 之前，添加以下代码，使得服务端退出后端口立即释放：

```python
tcp_server_socket.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, True)
```

- `socket.SOL_SOCKET`：表示当前套接字层
- `socket.SO_REUSEADDR`：端口复用选项
- `True`：启用端口复用

## 二、关键知识点速查表

| 知识点             | 关键内容                                                     |
| ------------------ | ------------------------------------------------------------ |
| **IP地址**         | 网络中设备的唯一标识，通过`ipconfig`/`ifconfig`查看          |

## 三、常见面试/考试题

1. **网络编程三要素是什么？各自的作用？**  
   IP地址（找到设备），端口号（找到进程），协议（定义通信规则）。

2. **TCP协议有哪些特点？**  
   面向连接、可靠传输、基于字节流。

3. **简述TCP三次握手的过程。**  
   - 客户端发送SYN（seq=x）  
   - 服务端回复SYN+ACK（seq=y, ack=x+1）  
   - 客户端回复ACK（ack=y+1）

4. **为什么TCP断开需要四次挥手？**  
   因为TCP连接是全双工的，双方都需要单独关闭自己的发送方向。

5. **什么是Socket？它在网络编程中扮演什么角色？**  
   Socket是进程间通信的工具，负责网络数据的传输，所有网络应用都基于Socket。

6. **TCP服务器端 `accept()` 返回的是什么？有什么作用？**  
   返回一个新的套接字和客户端地址。新套接字专门用于与该客户端收发数据。

7. **如何解决服务端程序退出后端口被占用的问题？**  
   设置端口复用：`setsockopt(SOL_SOCKET, SO_REUSEADDR, True)`

8. **服务端如何判断客户端已经断开连接？**  
   调用 `recv()` 返回的数据长度为0。

9. **网络传输中为什么要进行编码/解码？**  
   网络传输的数据必须是二进制格式（bytes），字符串需要编码后才能发送，接收到的二进制需要解码才能变成字符串。
