## 一、详细笔记

### 第1部分：面向对象的三大特性（概述）

面向对象编程（OOP）通常具有以下三大特性：

| 特性     | 定义                                                       | 白话理解                                         | 好处                 |
| -------- | ---------------------------------------------------------- | ------------------------------------------------ | -------------------- |
| **封装** | 将属性和方法书写到类的里面，可添加私有权限                 | 把属性和方法封装在一起，仅对外提供方法让别人访问 | 简化编程，增强安全性 |
| **继承** | 子类默认继承父类的所有属性和方法，可重写                   | 孩子可使用老爹的东西                             | 代码复用             |
| **多态** | 同一类事物具有多种形态，不同对象调用同一接口表现出不同状态 | 同样一个函数在不同场景下表现出不同形态           | 解耦合，可扩展       |

### 第2部分：Python中的封装

#### 2.1 封装的概念

- **定义**：在软件编程中，将**属性**和**方法**书写到类的里面的操作即为封装。
- **目的**：简化编程，增强安全性。使用者不必了解具体的实现细节，只需通过外部接口以特定的访问权限来使用类的成员。

---

#### 2.2 私有属性和私有方法

- **定义**：为属性和方法设置私有权限，使其**不继承给子类**，且**只能在类的内部使用**，不能在类的外部直接访问。
- **语法**：在属性名或方法名前面加上**双下划线 `__`**。
  - 私有属性：`self.__属性名`
  - 私有方法：`def __方法名(self):`

- **示例**：

  ```python
  class Girl():
      def __init__(self):
          self.name = '小美'      # 公有属性
          self.__age = 18         # 私有属性
      
      def __showinfo(self):       # 私有方法
          print('姓名：%s，年龄：%d' % (self.name, self.__age))
  
  girl = Girl()
  print(girl.name)        # 可正常访问
  # print(girl.__age)     # 报错：AttributeError
  # girl.__showinfo()     # 报错：AttributeError
  ```

---

#### 2.3 获取与设置私有属性值（getter / setter）

在Python中，一般定义`get_xx()`方法来获取私有属性，定义`set_xx()`方法来修改私有属性值。这样可以在方法中添加控制逻辑（如类型检查、范围验证）。

```python
class Girl():
    def __init__(self):
        self.name = '小美'
        self.__age = 18
    
    def get_age(self):
        return self.__age
    
    def set_age(self, age):
        if age < 0 or age > 150:
            print("年龄不合法")
            return
        self.__age = age

girl = Girl()
girl.set_age(19)
print(girl.get_age())   # 19
```

---

#### 2.4 封装的意义

- **私有属性的意义**：明确的区分内外，控制外部对隐藏属性的操作行为。例如，通过`set_age`可以限制年龄范围，防止无效数据。
- **私有方法的意义**：
  - “不让别人随便调”，保护对象的完整性，隐藏实现细节。
  - 简化调用代码，降低程序复杂度（将复杂流程封装在一个公有方法中，内部调用多个私有方法）。

**示例：ATM类（封装复杂流程）**

```python
class ATM:
    def __card(self):
        print('插卡')
    def __auth(self):
        print('用户认证')
    def __input(self):
        print('输入取款金额')
    def __take_money(self):
        print('取款')
    def __print_bill(self):
        print('打印账单')
    
    def withdraw(self):          # 对外公开的接口
        self.__card()
        self.__auth()
        self.__input()
        self.__take_money()
        self.__print_bill()

atm = ATM()
atm.withdraw()   # 用户只需调用这一个方法，内部细节被隐藏
```

**示例：People类（带类型检查的setter）**

```python
class People(object):  
    def __init__(self, name, age):  
        self.__name = name  
        self.__age = age  
  
    def tell_info(self):  
        print(f'Name:{self.__name} Age:{self.__age}')  
  
    def set_info(self, name, age):  
        if not isinstance(name, str):  
            print('名字必须是字符串类型')  
            return  
        if not isinstance(age, int):  
            print('年龄必须是数字类型')  
            return  
        self.__name = name  
        self.__age = age  
  
p = People('jack', 38)  
p.tell_info()  
p.set_info('jennifer', 18)  # 正确  
p.tell_info()  
p.set_info(123, 35)  # 类型错误，被拦截  
p.tell_info()  # 未修改成功
```

### 第3部分：Python中的继承

#### 3.1 继承的概念

- **继承**：一个类从另一个已有的类获得其成员的特性，就叫作继承。
- **派生**：从一个已有的类产生一个新的类，称为派生。继承和派生是从不同方向描述同一概念。
- **父类（基类）**：被继承的类。
- **子类（派生类/扩展类）**：继承后产生的新类。
- **扩展**：在子类中增加自己特有的特性。没有扩展，继承就没有意义。
- **基本语法**：

  ```python
  class 子类(父类):
      pass
  ```

- **注意**：在Python中，所有类默认继承`object`类，`object`是顶级类或基类。

---

#### 3.2 单继承

- **定义**：一个类只能继承自一个其他类（大多数面向对象语言的特性）。
- **案例**：Animal（吃、睡、叫） → Dog、Cat

```python
class Animal(object):
    def eat(self):
        print('吃...')
    def sleep(self):
        print('睡...')
    def call(self):
        print('叫...')

class Dog(Animal):
    pass

class Cat(Animal):
    pass

dog = Dog()
dog.eat()    # 继承自Animal
dog.sleep()
dog.call()
```

---

#### 3.3 重写父类属性和方法（覆盖）

- **定义**：当子类成员与父类成员名字相同时，子类成员会**覆盖**父类成员。通过子类实例访问时，真正起作用的是子类中定义的成员。
- **案例**：Dog和Cat重写`call()`方法，实现各自的叫声。

```python
class Dog(Animal):
    def call(self):
        print('汪汪叫...')

class Cat(Animal):
    def call(self):
        print('喵喵叫...')

dog = Dog()
dog.call()   # 输出：汪汪叫...
cat = Cat()
cat.call()   # 输出：喵喵叫...
```

> **注意**：父类的`call()`方法仍然存在，只是子类实例优先在自己的类中查找，找不到才去父类查找。

---

#### 3.4 调用父类属性和方法

当子类重写了`__init__`或其他方法时，常常需要显式调用父类的同名方法，以避免重复代码，并正确初始化父类定义的属性。

**方式1：直接通过父类名调用**（不推荐，需要手动传递self）

```python
class Dog(Animal):
    def __init__(self, name, age, sex):
        Animal.__init__(self, name, age)   # 手动传self
        self.sex = sex
```

**方式2：使用`super()`函数**（推荐）

- 完整写法：`super(当前类名, self).方法()`
- 简化写法：`super().方法()`

```python
class Dog(Animal):
    def __init__(self, name, age, sex):
        super().__init__(name, age)   # 调用父类的__init__
        self.sex = sex
    
    def __str__(self):
        return f'{self.name}，今年{self.age}岁了，我会汪汪叫...'

class Cat(Animal):
    def __init__(self, name, age, sex):
        super().__init__(name, age)
        self.sex = sex
    
    def __str__(self):
        return f'{self.name}，今年{self.age}岁了，我会喵喵叫...'
```

---

#### 3.5 多继承

- **定义**：一个类同时继承多个父类，并且具有所有父类的属性和方法。
- **语法**：`class Child(Father, Mother):`
- **MRO（Method Resolution Order，方法解析顺序）**：确定多继承时方法查找的顺序。可以通过`类名.__mro__`或`类名.mro()`查看。

  ```python
  class Father:
      pass
  class Mother:
      pass
  class Child(Father, Mother):
      pass
  print(Child.__mro__)   # 查看方法解析顺序
  ```

- **注意**：多继承可能带来命名冲突（多个父类有同名方法），Python按照MRO从左到右查找。使用时需谨慎。

---

#### 3.6 继承总结

| 问题                                     | 答案                           |
| ---------------------------------------- | ------------------------------ |
| 继承是什么？                             | 子类继承了父类的属性与方法     |
| 单继承是什么？                           | 一个类只继承一个父类           |
| 多继承是什么？                           | 一个类继承了多个父类           |
| 子类重写父类同名方法后，会优先调用哪个？ | 优先调用子类的方法（覆盖）     |
| 子类调用父类方法有几种方式？             | 两种：父类名.方法() 或 super() |

### 第4部分：Python中的多态

#### 4.1 多态的概念

- **定义**：多态指的是一类事物有多种形态。在Python中，多态是指**同一个函数，随着传入子类对象的不同，可以实现不同的功能**。
- **本质**：子类重写父类方法，调用不同子类对象的相同父类方法，产生不同的执行结果。
- **好处**：调用灵活，更容易编写出通用的代码，以适应需求的不断变化，提高代码的可扩展性。

---

#### 4.2 多态的实现步骤（三要素）

1. **有继承**：定义父类，定义子类，子类继承父类。
2. **函数重写**：子类重写父类的方法。
3. **父类引用指向子类对象**：将子类对象传给父类对象的调用者（即函数参数声明为父类类型，但实际传入子类对象）。

---

#### 4.3 多态案例1：动物叫

```python
class Animal(object):
    def __init__(self, name, age):
        self.name = name
        self.age = age
    def call(self):
        print(self.name, '会叫')

class Cat(Animal):
    def __init__(self, name, age, sex):
        super().__init__(name, age)
        self.sex = sex
    def call(self):
        print(self.name, '会“喵喵”叫')

class Dog(Animal):
    def __init__(self, name, age, sex):
        super().__init__(name, age)
        self.sex = sex
    def call(self):
        print(self.name, '会“汪汪”叫')

def do(animal):
    animal.call()      # 传入不同的animal对象，表现不同的call行为

a = Animal('小黑', 4)
c = Cat('喵喵', 2, '男')
d = Dog('旺财', 5, '女')

for x in (a, c, d):
    do(x)
```

**输出**：

```
小黑 会叫
喵喵 会“喵喵”叫
旺财 会“汪汪”叫
```

---

#### 4.4 多态案例2：工作犬

```python
class Dog(object):
    def work(self):
        print('指哪打哪...')

class ArmyDog(Dog):
    def work(self):
        print('追击敌人...')

class DrugDog(Dog):
    def work(self):
        print('追查毒品...')

class Person(object):
    def work_with_dog(self, dog):   # dog参数可以是任何有work方法的对象
        dog.work()

ad = ArmyDog()
dd = DrugDog()
police = Person()

police.work_with_dog(ad)   # 输出：追击敌人...
police.work_with_dog(dd)   # 输出：追查毒品...
```

> **动态语言的特性**：在Python中，调用实例方法时不检查类型，只要方法存在、参数正确就可以调用。因此，不一定要有严格的继承关系，只要对象有`work()`方法，就可以传入`work_with_dog`。这体现了**鸭子类型**。

---

#### 4.5 多态的优势

- 在不改变框架代码（如`work_with_dog`函数）的情况下，通过多态轻松实现模块和模块之间的**解耦合**。
- 实现了软件系统的**可扩展性**：新增一种狗（如`Corgi`），无需修改`Person`类，直接传入即可。

### 第5部分：面向对象的其他特性

#### 5.1 实例属性（对象属性）

- **定义**：属于具体对象实例的属性，每个对象有自己的独立副本。
- **定义位置**：通常在`__init__`方法中使用`self.属性 = 值`定义，也可以在类外部动态添加。
- **访问方式**：`对象名.属性名`（外部）或 `self.属性名`（内部）。

```python
class Phone(object):
    def __init__(self):
        self.brand = "华为"   # 实例属性

phone = Phone()
print(phone.brand)           # 华为
phone.color = "Yellow"       # 动态添加实例属性
print(phone.color)           # Yellow
```

---

#### 5.2 类属性

- **定义**：类所拥有的属性，被该类的**所有实例共享**。属于类本身，而不是某个实例。
- **定义位置**：直接在类内部、方法外部赋值。
- **访问方式**：推荐使用`类名.类属性`（清晰表明是类属性），也可以使用`对象名.类属性`（不推荐，容易混淆）。
- **用途**：记录与类相关的数据，如统计创建了多少个实例。

```python
class Person(object):
    count = 1          # 类属性

print(Person.count)    # 推荐：1
person = Person()
print(person.count)    # 也可以访问，但不推荐

# 统计工具对象个数
class Tool(object):
    count = 0
    def __init__(self, name):
        self.name = name
        Tool.count += 1

tool1 = Tool('斧头')
tool2 = Tool('榔头')
tool3 = Tool('铁锹')
print(Tool.count)      # 输出：3
```

---

#### 5.3 类方法

- **定义**：针对类对象定义的方法，可以直接访问类属性或调用其他类方法。
- **装饰器**：`@classmethod`
- **第一个参数**：`cls`，代表当前类（类似于实例方法的`self`）。通过`cls`可以访问类属性和其他类方法。
- **调用方式**：`类名.类方法名()`（不需要传递`cls`参数）。
- **使用场景**：需要操作类属性或类级别行为，且不需要访问实例属性。

```python
class Tool(object):
    count = 0
    
    @classmethod
    def show_tools_count(cls):
        print(f'工具对象的数量：{cls.count}')
    
    def __init__(self, name):
        self.name = name
        Tool.count += 1

Tool.show_tools_count()   # 输出：工具对象的数量：0
tool1 = Tool('斧头')
tool2 = Tool('榔头')
Tool.show_tools_count()   # 输出：工具对象的数量：2
```

---

#### 5.4 静态方法

- **定义**：既不需要访问实例属性/方法，也不需要访问类属性/方法的方法。它本质上是一个普通函数，但归属于类的命名空间。
- **装饰器**：`@staticmethod`
- **特点**：没有`self`或`cls`参数，不能访问实例属性和类属性。
- **调用方式**：`类名.静态方法名()`
- **使用场景**：实现与类相关但不需要访问类或实例任何属性的功能（如工具函数、辅助功能）。

```python
class Game(object):
    @staticmethod
    def show_help():
        print('游戏帮助信息：每个玩家每次玩一次总积分增加1分')

Game.show_help()   # 直接通过类名调用
```

---

#### 5.5 综合案例：Game游戏类设计

**需求**：

- 定义一个`Game`类
- 类属性`top_score`（或`all_score`）：记录游戏的历史总积分（所有玩家共享）
- 实例属性`player_name`：记录当前玩家的姓名
- 静态方法`show_help()`：显示游戏帮助信息
- 类方法`show_all_score()`：显示历史总积分
- 实例方法`start_game()`：开始当前玩家的游戏，每次调用总积分+1

**代码实现**：

```python
class Game(object):
    # 类属性：存储总积分
    all_score = 0
    
    def __init__(self, name):
        # 实例属性：玩家姓名
        self.name = name
    
    # 静态方法：显示帮助信息
    @staticmethod
    def show_help():
        print('游戏帮助信息：每个玩家每次玩一次总积分增加1分')
    
    # 类方法：显示历史总积分
    @classmethod
    def show_all_score(cls):
        print(f'当前总积分为：{cls.all_score}分')
    
    # 实例方法：玩家玩游戏
    def start_game(self):
        print(f"{self.name}玩家玩了一局，总积分+1")
        Game.all_score += 1

# 1. 显示帮助信息
Game.show_help()

# 2. 查看当前总积分
Game.show_all_score()

# 3. 创建玩家并开始游戏
wj1 = Game('张三')
wj1.start_game()
wj1.start_game()
Game.show_all_score()   # 积分应为2

wj2 = Game('李四')
wj2.start_game()
Game.show_all_score()   # 积分应为3
```

**输出示例**：

```
游戏帮助信息：每个玩家每次玩一次总积分增加1分
当前总积分为：0分
张三玩家玩了一局，总积分+1
张三玩家玩了一局，总积分+1
当前总积分为：2分
李四玩家玩了一局，总积分+1
当前总积分为：3分
```

## 二、关键知识点速查表

| 知识点           | 关键内容                                                     |
| ---------------- | ------------------------------------------------------------ |
| **封装**         | 将属性和方法放到类内部，私有成员用`__`前缀                   |
| **私有属性访问** | 定义`get_xx()`和`set_xx()`方法                               |
| **继承**         | `class 子类(父类):`，子类拥有父类所有非私有成员              |
| **单继承**       | 一个子类只有一个父类                                         |
| **多继承**       | `class 子类(父类1, 父类2):`，注意MRO顺序                     |
| **方法重写**     | 子类定义与父类同名的方法，会覆盖父类方法                     |
| **super()**      | 调用父类方法，推荐使用`super().方法()`                       |
| **多态**         | 同一方法传入不同子类对象产生不同结果；三要素：继承、重写、父类引用指向子类对象 |
| **实例属性**     | `self.属性`，每个对象独立                                    |
| **类属性**       | 类内部直接赋值，所有实例共享，推荐`类名.属性`访问            |
| **类方法**       | `@classmethod`，第一个参数`cls`，可访问类属性                |
| **静态方法**     | `@staticmethod`，无`self`/`cls`，相当于普通函数放在类中      |

## 三、常见面试/考试题

1. **面向对象的三大特性是什么？**  
   封装、继承、多态。

2. **私有属性和私有方法如何定义？它们能被继承吗？**  
   在名称前加双下划线`__`。私有属性和方法不会被继承，只能在类内部访问。

3. **如何强制访问私有属性？**  
   Python中可以通过`_类名__属性名`的方式访问（名称修饰），但强烈不建议这样做，破坏了封装性。

4. **`super()`的作用是什么？**  
   调用父类的方法，常用于在子类`__init__`中初始化父类定义的属性，避免重复代码。

5. **什么是MRO？如何查看？**  
   MRO是方法解析顺序，决定多继承时方法的查找顺序。通过`类名.__mro__`或`类名.mro()`查看。

6. **多态的实现条件是什么？**  
   - 继承（父类和子类）  
   - 子类重写父类方法  
   - 父类引用指向子类对象（将子类对象传入以父类为参数声明的函数）

7. **类属性和实例属性的区别？**  
   - 类属性属于类，所有实例共享，通过`类名.属性`访问。  
   - 实例属性属于具体对象，每个对象独立，通过`self.属性`或`对象.属性`访问。

8. **类方法和静态方法的区别？**  
   - 类方法有`@classmethod`装饰器，第一个参数是`cls`，可以访问类属性/方法。  
   - 静态方法有`@staticmethod`装饰器，没有`cls`或`self`参数，不能访问类或实例的任何属性，相当于普通函数。
