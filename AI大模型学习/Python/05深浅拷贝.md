# Python 深浅拷贝

---

## 一、详细笔记

### 第1部分：可变类型与不可变类型

在 Python 中，对象根据其值是否可以修改分为两类：**可变对象**和**不可变对象**。理解这两者的区别是掌握深浅拷贝的基础。

---

#### 1.1 不可变对象

- **定义**：一旦创建，其内存中的值就不能被改变的对象。
- **包含类型**：字符串（`str`）、元组（`tuple`）、数值类型（`int`、`float`、`bool`、`complex` 等）。
- **行为特点**：
  - 当尝试修改不可变对象时，Python 不会修改原有对象，而是创建一个**新的对象**，然后让变量指向这个新对象。
  - 原对象如果没有其他引用，会被垃圾回收。
- **示例**：

  ```python
  a = 10
  print(id(a))   # 假设输出 140736123456
  a = a + 1      # 实际上创建了新的整数对象 11
  print(id(a))   # 输出不同的地址，如 140736123488
  ```

- **本质**：修改变量实际上是让变量引用一个新的对象，原对象保持不变。

---

#### 1.2 可变对象

- **定义**：可以在不改变内存地址的情况下修改其内部值的对象。
- **包含类型**：列表（`list`）、字典（`dict`）、集合（`set`），以及用户自定义的可变类实例等。
- **行为特点**：
  - 修改可变对象的内容（如添加、删除、修改元素）时，对象本身的内存地址不变，即**原地修改**。
  - 不需要创建新对象，因此效率较高。
- **示例**：

  ```python
  lst = [1, 2, 3]
  print(id(lst))   # 输出一个地址
  lst.append(4)
  print(id(lst))   # 输出相同的地址
  ```

---

#### 1.3 为什么需要区分？

- 深浅拷贝的行为对于可变对象和不可变对象有显著差异。
- 不可变对象由于无法修改，拷贝时通常共享引用（不会产生副作用）。
- 可变对象拷贝时需要考虑是否独立修改，避免意外影响原数据。

---

### 第2部分：变量赋值执行原理

在 Python 中，变量不是存储值的“盒子”，而是指向对象的**引用**。

**赋值语句 `a = "python"` 执行过程**：

1. **创建变量**：在当前的命名空间（如全局作用域或函数局部作用域）中创建变量名 `a`。
2. **创建对象**：在堆内存中分配一块内存，存储字符串对象 `"python"`。
3. **建立引用**：将变量 `a` 与这个对象通过指针连接起来，即 `a` 引用该对象。

**关键理解**：

- 变量是对对象的**引用**（指针）。
- 多个变量可以引用同一个对象。
- 赋值操作 `b = a` 只是让 `b` 也引用 `a` 所引用的对象，不会创建对象的副本。

```python
a = [1, 2, 3]
b = a          # b 和 a 引用同一个列表对象
b.append(4)
print(a)       # [1, 2, 3, 4] —— a 也被修改了
```

---

### 第3部分：浅拷贝（Shallow Copy）

#### 3.1 概念

- **浅拷贝**：创建一个**新对象**，但其内部元素是原对象元素的**引用**（即只拷贝最外层对象本身，内层对象只拷贝引用）。
- 换句话说，浅拷贝复制了对象的顶层结构，但不会递归复制内部子对象。

---

#### 3.2 实现浅拷贝的方式

| 方式                      | 示例                                        |
| ------------------------- | ------------------------------------------- |
| 列表切片                  | `new_list = old_list[:]`                    |
| `list()` 构造函数         | `new_list = list(old_list)`                 |
| `copy` 模块的 `copy` 函数 | `import copy; new_obj = copy.copy(old_obj)` |
| 字典的 `copy()` 方法      | `new_dict = old_dict.copy()`                |
| 集合的 `copy()` 方法      | `new_set = old_set.copy()`                  |

---

#### 3.3 浅拷贝的行为分析

##### 3.3.1 对不可变类型进行浅拷贝

- **效果**：与**赋值**几乎一样。新变量与原变量引用同一个对象（内存地址相同）。
- **原因**：不可变对象不能被修改，所以共享引用是安全的，Python 不会浪费内存创建副本。

```python
import copy
a = "hello"
b = copy.copy(a)
print(id(a), id(b))   # 相同地址
```

---

##### 3.3.2 对可变类型进行浅拷贝（无嵌套结构）

- **情况**：原始对象只包含不可变类型的元素（如列表中的元素都是数字、字符串等）。
- **效果**：
  - 新对象与原对象的 **id 不同**（外层独立）。
  - 新对象与原对象的**内部元素指向相同的不可变对象**（但不可变对象不影响）。
  - 由于元素是不可变的，修改原对象的元素（如替换索引位置的值）不会影响新对象；反之亦然。
- **示例**：

  ```python
  import copy
  original = [1, 2, 3]
  shallow = copy.copy(original)
  print(id(original), id(shallow))   # 不同
  original[0] = 99
  print(original)   # [99, 2, 3]
  print(shallow)    # [1, 2, 3] —— 不受影响
  ```

---

##### 3.3.3 对可变类型进行浅拷贝（有嵌套结构）

- **情况**：原始对象中包含**可变子对象**（如列表中的元素也是列表、字典等）。
- **效果**：
  - 新对象的外层独立（id 不同）。
  - 新对象内部的子对象仍然是原对象子对象的**引用**（即共享内层可变对象）。
  - 如果修改**内层可变子对象**的内容，会同时影响原对象和新对象。
- **示例**：

  ```python
  import copy
  original = [1, [2, 3], 4]
  shallow = copy.copy(original)
  # 修改内层列表
  original[1].append(99)
  print(original)   # [1, [2, 3, 99], 4]
  print(shallow)    # [1, [2, 3, 99], 4] —— 也变了！
  
  # 但替换外层元素不影响对方
  original[0] = 100
  print(original)   # [100, [2, 3, 99], 4]
  print(shallow)    # [1, [2, 3, 99], 4] —— 外层独立
  ```

---

#### 3.4 浅拷贝总结

| 场景               | 外层对象 id | 内层对象关系       | 修改外层元素影响   | 修改内层可变元素影响 |
| ------------------ | ----------- | ------------------ | ------------------ | -------------------- |
| 不可变类型浅拷贝   | 相同        | 共享               | 不适用（不可修改） | 不适用               |
| 可变类型（无嵌套） | 不同        | 共享不可变子对象   | 不影响对方         | 不适用               |
| 可变类型（有嵌套） | 不同        | 共享可变子对象引用 | 不影响对方         | **互相影响**         |

---

### 第4部分：深拷贝（Deep Copy）

#### 4.1 概念

- **深拷贝**：递归地拷贝对象及其所有子对象，生成一个**完全独立**的副本。新对象与原对象在内存中完全隔离，修改任何一个都不会影响另一个。
- 深拷贝是**递归**的浅拷贝过程：对所有层级的可变对象都进行拷贝。

---

#### 4.2 实现深拷贝的方式

- 使用 `copy` 模块的 `deepcopy` 函数：

  ```python
  import copy
  new_obj = copy.deepcopy(old_obj)
  ```

---

#### 4.3 深拷贝的行为分析

##### 4.3.1 对可变类型进行深拷贝

- **无论嵌套多深**，所有层级的可变对象都会被复制，生成全新的独立对象。
- 新对象与原对象的 **id 完全不同**，且内部任何子对象的 id 也不同（对于可变子对象）。
- 修改原对象的任意层级（包括内层列表）不会影响新对象，反之亦然。

```python
import copy
original = [1, [2, 3], 4]
deep = copy.deepcopy(original)

print(id(original), id(deep))           # 不同
print(id(original[1]), id(deep[1]))     # 不同（内层列表也被拷贝）

original[1].append(99)
print(original)   # [1, [2, 3, 99], 4]
print(deep)       # [1, [2, 3], 4] —— 完全不受影响
```

---

##### 4.3.2 对不可变类型进行深拷贝

- **效果**：对于不可变类型，深拷贝不会创建新对象，而是直接拷贝引用（因为不可变对象无法修改，共享是安全的）。
- 这与浅拷贝、赋值的行为一致。

```python
import copy
a = (1, 2, 3)      # 元组不可变
b = copy.deepcopy(a)
print(id(a), id(b))   # 相同地址
```

> **注意**：如果不可变对象内部包含可变对象（如元组中包含列表），情况会不同。深拷贝会递归拷贝元组内的可变子对象，但元组本身仍然是不可变的（无法修改），不过新元组内的列表是独立的副本。实际上 `deepcopy` 会处理这种嵌套情况。

---

#### 4.4 深拷贝的注意事项

- **性能**：深拷贝需要递归复制所有层级，对于大型、深层嵌套的对象可能开销较大。
- **循环引用**：如果对象存在自引用或循环引用（如 `a = []; a.append(a)`），`deepcopy` 能正确处理，不会无限递归（内部维护了一个备忘录字典）。
- **不可拷贝对象**：某些对象无法被拷贝（如文件对象、数据库连接、线程锁等），对它们进行 `deepcopy` 会引发异常。

---

## 二、浅拷贝与深拷贝对比表

| 特性             | 浅拷贝 (shallow copy)            | 深拷贝 (deep copy)             |
| ---------------- | -------------------------------- | ------------------------------ |
| **复制层级**     | 仅复制最外层对象                 | 递归复制所有层级               |
| **内层可变对象** | 共享引用（修改内层会影响双方）   | 完全独立副本（互不影响）       |
| **不可变对象**   | 共享引用（id 相同）              | 共享引用（id 相同）            |
| **外层对象 id**  | 不同                             | 不同                           |
| **实现方式**     | `copy.copy()`、切片、`list()` 等 | `copy.deepcopy()`              |
| **性能开销**     | 小                               | 较大（递归）                   |
| **适用场景**     | 不需要独立内层对象的简单复制     | 需要完全独立副本，避免意外共享 |

---

## 三、关键代码示例汇总

```python
import copy

# 不可变类型
a = "hello"
b = copy.copy(a)        # 浅拷贝，id 相同
c = copy.deepcopy(a)    # 深拷贝，id 也相同
print(id(a), id(b), id(c))   # 全部相同

# 可变类型无嵌套
lst1 = [1, 2, 3]
lst2 = copy.copy(lst1)   # 浅拷贝，外层独立
lst1[0] = 99
print(lst1)   # [99, 2, 3]
print(lst2)   # [1, 2, 3]

# 可变类型有嵌套
lstA = [1, [2, 3], 4]
lstB = copy.copy(lstA)   # 浅拷贝
lstA[1].append(99)       # 修改内层列表
print(lstA)   # [1, [2, 3, 99], 4]
print(lstB)   # [1, [2, 3, 99], 4] —— 受影响

lstC = copy.deepcopy(lstA)   # 深拷贝
lstA[1].append(100)
print(lstA)   # [1, [2, 3, 99, 100], 4]
print(lstC)   # [1, [2, 3, 99], 4] —— 不受影响
```

## 五、常见面试/考试题

1. **什么是浅拷贝？什么是深拷贝？**
   - 浅拷贝：只复制最外层对象，内部元素是原对象的引用。
   - 深拷贝：递归复制所有层级的对象，生成完全独立的副本。

2. **浅拷贝在什么情况下会影响原对象？**
   - 当原对象包含可变子对象（如嵌套列表、字典），且修改了该可变子对象的内容时，浅拷贝的副本也会受到影响。

3. **深拷贝是否会复制不可变对象？**
   - 不会。对于不可变对象，深拷贝直接共享引用，因为不可变对象无法被修改，共享是安全的。

4. **如何实现浅拷贝和深拷贝？**
   - 浅拷贝：`copy.copy()`、列表切片、`list()` 构造等。
   - 深拷贝：`copy.deepcopy()`。

5. **为什么有时候深拷贝比浅拷贝消耗更多资源？**
   - 深拷贝需要递归复制所有嵌套对象，可能产生大量新对象，内存和时间开销更大。

6. **赋值（`b = a`）与浅拷贝（`b = copy.copy(a)`）有何区别？**
   - 赋值：`b` 和 `a` 引用**同一个对象**（id 相同）。
   - 浅拷贝：`b` 是一个**新对象**（外层 id 不同），但内部元素共享引用。
