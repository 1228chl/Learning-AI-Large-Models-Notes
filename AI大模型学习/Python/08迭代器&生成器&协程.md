# Python 迭代器、生成器、协程

---
## 一、详细笔记

### 第1部分：迭代器（Iterator）

#### 1.1 什么是迭代器

- **定义**：迭代器是 Python 中的一种对象，用于在数据集合中**逐个访问元素**，而不需要暴露数据集合的底层实现。它提供了一种遍历集合元素的标准方式。
- **适用数据结构**：列表、元组、字符串、字典、集合，以及 `range()` 等任何支持迭代的数据结构。实际上，`range()` 返回的就是一个迭代器（更准确说是可迭代对象，但它的迭代器行为类似）。
- **核心协议**：一个对象要成为迭代器，必须实现以下两个方法：
  - `__iter__()`：返回迭代器对象自身（通常直接返回 `self`）。
  - `__next__()`：返回序列中的下一个元素。如果没有元素了，则抛出 `StopIteration` 异常。

#### 1.2 迭代器的特点

| 特点       | 说明                                                                  |
| -------- | ------------------------------------------------------------------- |
| **手动管理** | 需要显式地实现 `__iter__()` 和 `__next__()` 方法，自己控制迭代逻辑                     |
| **状态管理** | 迭代器自己维护当前位置、结束条件等状态                                                 |
| **内存使用** | 通常采用**惰性计算**（lazy evaluation），即按需生成数据，而不是一次性把所有数据加载到内存。这使得迭代器非常节省内存 |

#### 1.3 迭代器底层实现示例

```python
class MyIterator:
    def __init__(self, start, end):
        self.current = start   # 当前值
        self.end = end         # 结束值（不包含）

    def __iter__(self):
        return self            # 返回自身作为迭代器

    def __next__(self):
        if self.current >= self.end:
            raise StopIteration   # 迭代结束，抛出异常
        value = self.current
        self.current += 1
        return value

# 使用迭代器
iterator = MyIterator(1, 5)
for num in iterator:
    print(num)   # 输出 1,2,3,4
```

**代码解析**：
- `__init__` 初始化起始和结束值。
- `__iter__` 返回自身，这样迭代器就可以直接用于 `for` 循环（`for` 会先调用 `__iter__` 获取迭代器对象）。
- `__next__` 每次返回当前值，并将计数器加1；当超过结束范围时抛出 `StopIteration`，`for` 循环捕获该异常并正常退出。

#### 1.4 迭代器的优势

- **惰性计算**：只在需要时才计算下一个值，适合处理**大型数据集合**或**流数据**（如文件读取、网络数据流）。
- **统一遍历接口**：无论底层数据结构如何，都可以使用相同的 `for` 循环语法遍历。

### 第2部分：生成器（Generator）

#### 2.1 什么是生成器

- **定义**：生成器是一种**根据程序员制定的规则循环生成数据**的机制。它不是一次性生成所有数据，而是**使用一个，生成一个**，从而**节约大量内存**。
- **本质**：生成器是一种特殊的迭代器，但实现更简洁。

#### 2.2 创建生成器的两种方式

##### 方式1：生成器推导式（生成器表达式）

- **语法**：`(表达式 for 变量 in 可迭代对象 [if 条件])`
- **注意**：使用**圆括号** `()`，不是元组推导式（元组推导式实际上就是生成器表达式）。圆括号内的内容定义了数据生成的规则，返回的是一个生成器对象，对象内部存储的不是数据，而是**生成规则**。

```python
# 创建生成器：生成 0~4 每个数乘以2的结果
my_generator = (i * 2 for i in range(5))

print(my_generator)   # <generator object <genexpr> at 0x...>

# 方法1：使用 next() 逐个获取
print(next(my_generator))   # 0
print(next(my_generator))   # 2
print(next(my_generator))   # 4

# 方法2：使用 for 循环遍历（推荐，自动处理 StopIteration）
for value in my_generator:
    print(value)   # 继续输出 6, 8
```

##### 方式2：yield 关键字

- 在函数内部使用 `yield` 关键字，该函数就不再是普通函数，而是一个**生成器函数**。调用生成器函数会返回一个生成器对象，不会立即执行函数体。
- `yield` 的特性：
  1. **返回**：将 `yield` 后面的值返回给调用者（类似于 `return`）。
  2. **暂停**：在 `yield` 处**暂停**函数的执行，保存当前所有状态（局部变量、指令指针等）。
  3. **恢复**：下次通过 `next()` 或 `for` 循环获取值时，从暂停的位置**继续执行**。

| return    | yield            |
| --------- | ---------------- |
| 结束函数，永久退出 | 返回值，**暂停函数，不结束** |
| 只能返回一次    | 可以多次返回值          |
| 普通函数      | 生成器函数            |
| 立刻返回结果    | 惰性加载，按需要生成       |


```python
def my_generator(n):
    for i in range(n):
        print('开始生成...')
        yield i
        print('完成一次...')

if __name__ == '__main__':
    g = my_generator(2)   # 获得生成器对象，此时函数并未执行
    # 使用 next() 获取
    print(next(g))   # 打印：开始生成...  然后返回 0；打印：开始生成...
    print(next(g))   # 从上次暂停处继续：打印"完成一次..."，然后进入下一次循环：打印"开始生成..."，返回 1
    # 第三次调用 next(g) 会抛出 StopIteration
```

**输出**：
```
开始生成...
0
完成一次...
开始生成...
1
```

> **重要注意点**：
> - 生成器生成完数据后，再次调用 `next()` 会抛出 `StopIteration` 异常。
> - 使用 `for` 循环遍历生成器时，`for` 内部自动处理了 `StopIteration` 异常，因此更推荐使用 `for`。
> - `while` 循环遍历时需要手动捕获异常。

#### 2.3 生成器的应用场景 – 数据加载器（Dataloader）封装

在机器学习和深度学习中，数据集通常很大，无法一次性加载到内存。需要**按批次（batch）** 读取数据。生成器非常适合实现这种**数据迭代器**。

```python
import math

def dataset_loader(batch_size):
    # 1. 读取文件（假设有一个歌词文件）
    with open('./jaychou_lyrics.txt', 'r') as file:
        lines = file.readlines()
    
    # 2. 统计总数据量
    lyrics_number = len(lines)
    
    # 3. 计算总批次数（向上取整）
    batch_number = math.ceil(lyrics_number / batch_size)
    
    # 4. 遍历每一个 batch，yield 返回切片
    for idx in range(batch_number):
        start = idx * batch_size
        end = idx * batch_size + batch_size
        yield lines[start:end]

if __name__ == '__main__':
    dataloader = dataset_loader(8)   # 每批 8 条数据
    for data in dataloader:
        print(data)   # 逐批输出数据，内存中只保存当前批次
```

**优点**：无论数据集多大，内存中始终只保存一个批次的数据，极大地节省内存。

#### 2.4 生成器 vs 迭代器 对比

| 对比维度       | 迭代器                                                   | 生成器                                         |
| -------------- | -------------------------------------------------------- | ---------------------------------------------- |
| **实现方式**   | 需要手动实现 `__iter__()` 和 `__next__()` 方法，管理状态 | 使用 `yield` 关键字，Python 自动管理状态       |
| **代码复杂度** | 代码较多，需要处理边界和异常                             | 简洁，类似普通函数                             |
| **性能与内存** | 通常也是惰性计算，但手动实现可能出错                     | 自动惰性计算，内存友好                         |
| **使用场景**   | 需要高度自定义迭代过程（如遍历树结构）                   | 按需生成序列数据（如读取大文件、生成无限序列） |

### 第3部分：协程（Coroutine）

#### 3.1 协程的概念

- **发展历程**：
  - Python 2.5：引入 `yield` 关键字，可以实现简单的协程（通过 `send()` 方法）。
  - Python 3.3：引入 `yield from` 语法，简化了嵌套生成器和协程。
  - Python 3.5：正式推出 **`async` / `await`** 语法，使协程成为一等公民，但底层依然基于生成器机制。
- **定义**：协程是**用户态的轻量级线程**，由**程序员控制调度**，而非操作系统。它可以在执行过程中**主动挂起**（使用 `await`），让出 CPU 去执行其他任务，稍后再恢复执行。
- **生活比喻**：想象一个餐厅**服务员（单线程）** 同时服务多桌客人。当 A 桌客人在看菜单时（I/O 等待），服务员不去干等，而是去 B 桌点菜；B 桌在思考时，又去 C 桌结账。通过**主动让出和恢复**，一个服务员高效服务多桌，而不是为每桌配一个服务员（多线程）。

#### 3.2 协程的作用

- **极高的并发性能**：单线程中可以支撑**数万个协程**同时存在，而线程通常只能创建数千个（受内存和操作系统限制）。
- **极低的切换开销**：协程切换不涉及操作系统内核，只需保存和恢复少量寄存器，开销远小于线程上下文切换（微秒级 vs 微秒到毫秒级）。
- **完美的 I/O 并发解决方案**：当协程遇到 I/O 操作（网络请求、文件读写、数据库查询）时，会自动挂起，让出 CPU 给其他协程，等 I/O 完成后再恢复。这样 CPU 几乎不会空闲等待。

#### 3.3 协程三要素

| 要素              | 说明                                                         |
| ----------------- | ------------------------------------------------------------ |
| **async def**     | 定义一个协程函数。调用它不会立即执行，而是返回一个协程对象   |
| **await**         | 在协程内部，使用 `await` 挂起当前协程，等待一个可等待对象（另一个协程、`asyncio.Future` 或 `asyncio.Task`）完成。`await` 是**主动让出控制权**的点 |
| **asyncio.run()** | 启动事件循环（event loop），执行协程的最常用顶层入口。事件循环负责调度所有协程 |

#### 3.4 协程代码示例
- 步骤
	- **导入 asyncio 库**：提供协程事件循环、sleep 异步等待功能
    - **async def 定义协程函数**：声明异步函数，调用不会立即执行
    - **await 异步等待任务**：遇到等待主动暂停、让出 CPU，实现异步并发
    - **并发执行多个协程：**使用 `asyncio.gather(*tasks)` 同时调度多个任务
    - **asyncio.run ()启动事件循环**：使用 `asyncio.run(main())` 作为程序入口

```python
import asyncio
import time

async def worker(name, delay):
    """
    模拟一个异步工作单元。
    :param name: 任务名称
    :param delay: 模拟的耗时秒数
    :return: 任务完成后的结果字符串
    """
    print(f"[协程] {name} 开始，将耗时 {delay} 秒")
    # await asyncio.sleep(delay) 模拟非阻塞 I/O 操作
    # 在等待期间，事件循环会切换到其他协程
    await asyncio.sleep(delay)
    print(f"[协程] {name} 完成！")
    return f"{name} 的结果"

async def main():
    start = time.time()
    
    # 创建多个协程对象（此时尚未执行）
    tasks = [
        worker("任务A", 2),
        worker("任务B", 1),
        worker("任务C", 3),
    ]
    
    # 并发调度所有任务，并等待它们全部完成
    # asyncio.gather 会同时运行这些协程，总耗时 ≈ max(2,1,3) = 3 秒
    results = await asyncio.gather(*tasks)
    
    print(f"结果: {results}")
    print(f"总耗时: {time.time() - start:.2f} 秒\n")

if __name__ == "__main__":
    asyncio.run(main())   # 启动事件循环，运行 main 协程
```

**输出**（顺序可能略有不同，但总耗时约 3 秒）：
```
[协程] 任务A 开始，将耗时 2 秒
[协程] 任务B 开始，将耗时 1 秒
[协程] 任务C 开始，将耗时 3 秒
[协程] 任务B 完成！
[协程] 任务A 完成！
[协程] 任务C 完成！
结果: ['任务A 的结果', '任务B 的结果', '任务C 的结果']
总耗时: 3.00 秒
```

**关键点**：
- 三个任务**几乎同时开始**，而不是顺序执行。
- 总耗时取决于最慢的任务（3秒），而不是总和（6秒）。
- `await asyncio.sleep(delay)` 模拟 I/O 等待，此时协程挂起，事件循环去运行其他协程。

| 写法                     | 效果   | 协程能否切换 | 作用                     |
| ---------------------- | ---- | ------ | ---------------------- |
| time.sleep(2)          | 阻塞   | 不能     | 卡死整个线程，CPU 空转，其他协程全停   |
| await asyncio.sleep(2) | 异步等待 | 能      | **让出 CPU**，去跑别的协程，实现并发 |

#### 3.5 协程 vs 线程 vs 进程 对比与选择指南

| 特性         | 进程                             | 线程                            | 协程                                                    |
| ------------ | -------------------------------- | ------------------------------- | ------------------------------------------------------- |
| **调度者**   | 操作系统                         | 操作系统                        | 用户程序（事件循环）                                    |
| **切换开销** | 大（需要切换内存空间、CPU 状态） | 中等                            | 极小（只保存少量寄存器）                                |
| **内存占用** | 独立内存空间，较大               | 共享进程内存，较小              | 极轻量，一个线程可容纳数万个协程                        |
| **数据同步** | 需要 IPC（队列、管道等）         | 需要锁（容易出错）              | 无需锁（单线程内串行，不会同时修改）                    |
| **并行能力** | 可以真正并行（多核）             | 受 GIL 限制，计算密集型不能并行 | 不能并行（单线程），但可以并发                          |
| **适用场景** | CPU 密集型、需要稳定隔离         | 简单的 I/O 并发                 | **高并发 I/O 密集型**（网络爬虫、Web 服务器、聊天服务） |

**选择指南**：
- **CPU 密集型并行计算** → 使用**多进程**（`multiprocessing`）
- **高并发 I/O 密集型计算**（数万连接） → 使用**协程**（`asyncio`）
- **简单并发 I/O 密集型计算**（几十到几百连接） → 使用**多线程**（`threading`）

> **补充说明**：
> **CPU 密集型**：任务的执行时间主要取决于 CPU 的处理速度，大部分时间在进行数学运算、逻辑判断、数据处理等计算操作。例如：视频编码、科学计算。
> **I/O 密集型**：任务的执行时间主要取决于输入/输出操作的等待时间，如网络请求、磁盘读写、数据库查询。CPU 大部分时间处于空闲等待状态。例如：网络爬虫、Web 服务器、文件处理。

#### 3.6 协程使用步骤总结

1. **定义协程函数**：使用 `async def` 定义。
2. **在协程内部标记等待点**：使用 `await` 挂起协程，等待 I/O 操作或子协程完成。
3. **并发执行多个协程**：使用 `asyncio.gather(*tasks)` 同时调度多个任务。
4. **启动事件循环**：使用 `asyncio.run(main())` 作为程序入口。

## 二、关键知识点速查表

| 知识点               | 关键内容                                                     |
| -------------------- | ------------------------------------------------------------ |
| **迭代器**           | 实现 `__iter__` 和 `__next__` 的对象，惰性计算，抛出 `StopIteration` 结束 |
| **生成器**           | 使用 `yield` 的函数，或生成器表达式 `(expr for ...)`，自动管理状态 |
| **yield 特性**       | 返回值 + 暂停函数 + 恢复执行                                 |
| **生成器 vs 迭代器** | 生成器是简化版的迭代器，代码更简洁                           |
| **协程**             | `async def` 定义，`await` 挂起，`asyncio.run()` 启动         |
| **协程作用**         | 极高并发、极低开销、完美 I/O 密集型任务                      |
| **asyncio.gather**   | 并发执行多个协程，等待全部完成                               |
| **asyncio.sleep**    | 模拟非阻塞 I/O，挂起当前协程                                 |
| **并发 vs 并行**     | 并发是逻辑上同时（单核也可），并行是物理上同时（多核）       |

## 三、常见面试/考试题

1. **什么是迭代器？如何自定义迭代器？**  
   迭代器是实现了 `__iter__` 和 `__next__` 方法的对象。自定义时，`__next__` 返回下一个值，结束时抛出 `StopIteration`。

2. **生成器有哪些优点？**  
   节约内存（惰性计算），代码简洁，适合处理大数据流。

3. **`yield` 关键字的作用是什么？**  
   - 返回一个值给调用者  
   - 暂停函数执行，保存状态  
   - 下次调用 `next()` 时从暂停处继续执行

4. **生成器和迭代器的区别？**  
   生成器是使用 `yield` 或生成器表达式创建的简化迭代器，自动管理状态；迭代器需要手动实现两个方法。

5. **什么是协程？协程与线程有什么区别？**  
   协程是用户态轻量级线程，由程序调度，切换开销极小。线程由操作系统调度，切换开销较大，且多线程需要处理锁竞争。

6. **协程适合什么场景？**  
   高并发的 I/O 密集型任务，如网络爬虫、Web 服务器、实时聊天系统。

7. **如何并发运行多个协程？**  
   使用 `asyncio.gather(*tasks)` 或 `asyncio.wait(tasks)`。

8. **`await` 的作用是什么？**  
   挂起当前协程，让出控制权给事件循环，等待被等待对象（协程、Task、Future）完成。

9. **Python 的协程能利用多核 CPU 吗？**  
   不能。协程本质是单线程的，要利用多核需结合多进程（如 `asyncio` + `multiprocessing`）。
