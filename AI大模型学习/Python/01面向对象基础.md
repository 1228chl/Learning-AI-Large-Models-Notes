# Python 面向对象基础 

---

## 一、详细笔记

### 第1部分：什么是面向对象（编程思想）

#### 1.1 编程思想概述

- **定义**：编程思想是人们利用计算机解决实际问题时的一种思维方式。
- **通用性**：编程思想与计算机语言本身没有太大关系，是通用的。
- **Python的特点**：Python是**同时支持面向过程和面向对象**的编程语言（多范式语言）。

#### 1.2 面向过程思想

- **定义**：解决问题时，把所需要的步骤都列出来，然后按照步骤写代码挨个实现。这是一种过程化的叙事思维。
- **核心**：以过程（函数）为中心，数据与操作分离。
- **适用场景**：
  - 数据转换脚本
  - 简单的算法实现
  - 一次性工具脚本

#### 1.3 面向对象思想

- **定义**：以对象为中心的编程范式。对象是**数据（属性）** 和**操作数据的方法**的封装体。程序通过对象之间的交互来完成任务。
- **核心思想**：**程序 = 对象 + 消息传递**
- **特点**：
  - 将数据和操作数据的方法绑定在一起
  - 通过**类**和**对象**来组织代码
  - 宗旨是**模拟现实世界**
  - **一切皆对象**
- **优点**：使编程与实际世界更加接近，所有对象被赋予属性和方法，编程更加富有人性化。
- **适用场景**：
  - 复杂的业务逻辑
  - 需要建模现实世界的系统
  - 大型项目，多人协作
  - 需要良好扩展性的框架

#### 1.4 面向过程 vs 面向对象 对比

| 对比项     | 面向过程         | 面向对象               |
| ---------- | ---------------- | ---------------------- |
| 视角       | 步骤过程         | 操作对象               |
| 中心       | 函数/过程        | 对象（数据+方法）      |
| 数据与操作 | 分离             | 封装在一起             |
| 本质       | 编程基础         | 对面向过程的再次封装   |
| 适用       | 简单、一次性任务 | 复杂、大型、可扩展系统 |

> **小练习**：举生活中的案例（如“做饭”），分别用面向过程和面向对象思维分析。

### 第2部分：面向对象的基本概念

#### 2.1 类（Class）

- **概念**：对现实事物的**抽象描述**，是创建对象的模板/图纸。
- **特点**：看不见、摸不着，是抽象的概念模型。
- **生活类比**：汽车图纸——它规定了汽车应该具有哪些部件和行为（如“跑起来”），但图纸本身不是汽车。
- **定义语法**：

  ```python
  class 类名:
      # 定义方法
      方法列表...
  ```

- **示例**：

  ```python
  class Car:
      def run(self):
          print("能跑起来了...")
  ```

#### 2.2 对象（Object）

- **概念**：现实事物的**具体体现**，是根据类创建出来的实体（实例）。
- **特点**：看得见、摸得着。
- **生活类比**：根据汽车图纸生产出来的能跑起来的汽车实体。
- **创建语法**：

  ```python
  对象名 = 类名()
  ```

- **调用方法**：

  ```python
  对象名.方法名()
  ```

- **示例**：

  ```python
  car = Car()        # 创建对象
  car.run()          # 调用方法
  ```

#### 2.3 self 关键字

- **概念**：Python内置的关键字，用于指向**对象实例本身**。
- **作用**：
  - 区分是哪个对象调用了类的方法（一个类可以实例化多个对象）
  - 在类内部访问对象的属性和其他方法
- **验证**：打印对象名和`self`的值，两者指向相同的内存地址。
- **在类内部调用方法**：`self.方法名()`

```python
class Car:
    def run(self):
        print(f"self的值:{self}")
    def work(self):
        self.run()      # 类内部调用其他方法

car = Car()
print(f"car对象:{car}")
car.work()
```

### 第3部分：添加和获取对象属性

#### 3.1 属性的概念

- **属性**：表示对象的**固有特征**，在Python中使用**变量**表示。
- **示例**：人的姓名、年龄、身高、体重等，都是对象的属性。

#### 3.2 类外部添加和获取属性

- **添加属性**：`对象名.属性名 = 属性值`
- **获取属性**：`对象名.属性名`

```python
class Car:
    def run(self):
        print("能跑起来了...")

car = Car()
car.color = "红色"    # 添加属性
car.number = 4
print(f"车的颜色:{car.color}")   # 获取属性
print(f"轮胎数:{car.number}")
```

#### 3.3 类内部获取对象属性

- 在类的方法内部，通过 `self.属性名` 来获取属性值。

```python
class Car:
    def run(self):
        print("能跑起来了...")
    def show(self):
        print(f"车的颜色为:{self.color}")
        print(f"车的轮胎个数为:{self.number}")

car = Car()
car.color = "红色"
car.number = 4
car.show()    # 类内部获取属性
```

### 第4部分：魔法方法

#### 4.1 魔法方法的概念

- **定义**：给Python类增加魔力的特殊方法，名称被**双下划线所包围**（如 `__init__`）。
- **特点**：在特殊情况下会被**自动调用**，不需要开发者手动调用。
- **格式**：`__方法名__()`

#### 4.2 `__init__()` 初始化方法

- **调用时机**：当**新创建一个对象时**，Python解释器会自动触发 `__init__()` 方法。
- **作用**：初始化对象的属性（在对象创建时就赋予属性值）。

**无参数形式**：在方法内部直接赋值固定值。

```python
class Car:
    def __init__(self):
        self.color = "Black"
        self.number = 3
    def show(self):
        print(f"车的颜色为:{self.color}, 轮胎数:{self.number}")

car = Car()
car.show()   # 车的颜色为:Black, 轮胎数:3
```

**有参数形式**：通过外部传递参数来初始化属性值。

```python
class Car:
    def __init__(self, color, number):
        self.color = color
        self.number = number

car = Car("Red", 4)
print(car.color)   # Red
print(car.number)  # 4
```

> **执行顺序**：`Car("Red",4)` → 自动调用 `__init__(self,"Red",4)` → 初始化属性 → 返回对象。

#### 4.3 `__str__()` 字符串表示方法

- **调用时机**：使用 `print(对象)` 输出对象时自动调用。
- **作用**：返回对象的字符串表示，替代默认的内存地址输出。
- **要求**：必须返回**字符串类型**。

```python
class Car:
    def __init__(self, color, number):
        self.color = color
        self.number = number
    def __str__(self):
        return f"车的颜色:{self.color}, 轮胎个数:{self.number}"

car = Car("Red", 4)
print(car)   # 输出：车的颜色:Red, 轮胎个数:4
```

> **对比**：如果没有 `__str__` 方法，`print(car)` 会输出类似 `<__main__.Car object at 0x7f8a1c0b4a90>` 的内存地址。

#### 4.4 `__del__()` 析构方法

- **调用时机**：当**删除对象**时自动调用（使用 `del` 删除对象，或文件执行结束对象被销毁时）。
- **作用**：释放对象占用的资源（如关闭文件、释放网络连接等）。

```python
class Car:
    def __init__(self, brand):
        self.brand = brand
    def __del__(self):
        print("__del__()方法被自动调用了，对象即将被销毁")

car = Car("BMW")
print(car.brand)   # BMW
del car            # 删除对象，自动调用 __del__
# print(car.brand)   # 报错：NameError: name 'car' is not defined
```

### 第5部分：面向对象的综合案例

#### 5.1 案例1：减肥

**需求描述**：

- 小明同学当前体重是100 kg。
- 每当他**跑步**一次时，体重减少0.5 kg。
- 每当他**大吃大喝**一次时，体重增加2 kg。
- 采用面向对象方式实现。

**代码实现**：

```python
class Student:
    def __init__(self):
        """初始化体重为100kg"""
        self.current_weight = 100
    
    def run(self):
        """跑步一次，体重减0.5kg"""
        self.current_weight -= 0.5
        print(f"跑步一次,减少0.5kg,当前体重为{self.current_weight}kg.")
    
    def eat(self):
        """大吃大喝一次，体重增2kg"""
        self.current_weight += 2
        print(f"大吃大喝一顿,增加2kg,当前体重为:{self.current_weight}kg.")

# 创建对象
student = Student()
# 模拟行为
student.run()    # 跑步一次
student.eat()    # 大吃大喝一次
```

**输出示例**：

```
跑步一次,减少0.5kg,当前体重为99.5kg.
大吃大喝一顿,增加2kg,当前体重为:101.5kg.
```

#### 5.2 案例2：烤地瓜

**需求描述**：

- 地瓜有**生熟状态**和**添加的调料**。
- 用户根据意愿设定每次烤地瓜的时间，地瓜被烤的总时间决定生熟状态。
- 用户可以添加多种调料，调料被存储起来。
- 当输出对象时，显示地瓜被烤的总时长、当前生熟状态、已经添加的调料列表。

**状态规则**：

| 被烤总时间   | 生熟状态 |
| ------- | ---- |
| 0 ~ 3分钟 | 生的   |
| 3 ~ 5分钟 | 半生不熟 |
| 5 ~ 8分钟 | 熟了   |
| ≥ 8分钟   | 烤糊了  |

**代码实现**：

```python
class SweetPotato:
    def __init__(self):
        self.cook_time = 0        # 被烤的总时间
        self.cook_state = '生的'  # 烤的状态
        self.condiments = []      # 调料列表
    
    def cook(self, time):
        """烤地瓜，time为本次烤的分钟数"""
        self.cook_time += time
        
        # 根据总时间判断生熟状态
        if 0 <= self.cook_time < 3:
            self.cook_state = '生的'
        elif 3 <= self.cook_time < 5:
            self.cook_state = '半生不熟'
        elif 5 <= self.cook_time < 8:
            self.cook_state = '熟了'
        elif self.cook_time >= 8:
            self.cook_state = '烤糊了'
    
    def add_condiments(self, condiment):
        """添加调料"""
        self.condiments.append(condiment)
    
    def __str__(self):
        return f'这个地瓜的被烤过的时间是{self.cook_time}, 状态是{self.cook_state}, 调料有{self.condiments}'

# 创建对象并模拟
digua = SweetPotato()
print(digua)                # 初始状态

digua.cook(10)              # 烤10分钟
digua.add_condiments('白糖')
print(digua)                # 烤糊了，加白糖

digua.cook(2)               # 再烤2分钟（总时间12分钟）
digua.add_condiments('辣椒面')
print(digua)                # 仍然是烤糊了，调料增加
```

**输出示例**：

```
这个地瓜的被烤过的时间是0, 状态是生的, 调料有[]
这个地瓜的被烤过的时间是10, 状态是烤糊了, 调料有['白糖']
这个地瓜的被烤过的时间是12, 状态是烤糊了, 调料有['白糖', '辣椒面']
```

## 二、关键知识点速查表

| 知识点            | 关键内容                   |
| -------------- | ---------------------- |
| **类**          | 抽象模板，定义属性和行为           |
| **对象**         | 类的具体实例                 |
| **属性**         | 对象的特征，用变量表示            |
| **方法**         | 对象的行为，用函数表示            |
| **self**       | 指向对象实例本身，用于区分不同对象      |
| **`__init__`** | 初始化方法，创建对象时自动调用        |
| **`__str__`**  | 字符串表示方法，print(对象)时自动调用 |
| **`__del__`**  | 析构方法，删除对象时自动调用         |
| **类外添加属性**     | `对象.属性 = 值`            |
| **类外获取属性**     | `对象.属性`                |
| **类内获取属性**     | `self.属性`              |

## 三、常见面试/考试题

1. **什么是面向对象？与面向过程的区别？**  
   面向对象是以对象为中心，封装数据和方法；面向过程是以过程/函数为中心。面向对象更接近现实世界思维。

2. **类和对象的关系？**  
   类是模板，对象是根据模板创建的具体实例。一个类可以创建多个对象。

3. **self 的作用是什么？**  
   指向当前对象实例本身，用于在类内部访问属性和方法，区分不同对象。

4. **`__init__` 方法什么时候执行？有什么作用？**  
   创建对象时自动执行，用于初始化对象的属性。

5. **`__str__` 方法什么时候执行？**  
   使用 `print(对象)` 时自动执行，返回对象的字符串表示。

6. **`__del__` 方法什么时候执行？**  
   对象被删除（`del` 或程序结束）时自动执行，用于释放资源。

7. **如何在类外部给对象添加属性？**  
   `对象名.属性名 = 属性值`

8. **如何在类内部获取对象的属性？**  
   `self.属性名`
