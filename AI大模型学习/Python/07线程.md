# Python 线程

---

## 一、详细笔记

### 第1部分：线程的介绍

#### 1.1 线程的概念

- **进程**：操作系统**分配资源**的基本单位。一旦创建一个进程，系统就会为其分配一定的资源（内存、文件句柄等）。
- **线程**：**CPU调度**的基本单位。每个进程至少有一个线程，这个线程被称为**主线程**。线程本身不拥有系统资源，而是共享所属进程的资源。

> 形象比喻：进程就像一个工厂（拥有厂房、设备等资源），线程就像工厂里的工人（使用资源干活，是真正干活的人）。

#### 1.2 线程的作用

- 线程**依附于进程**执行，不能独立存在。
- 线程用于**实现多任务编程**。在一个进程中，可以创建多个线程，让它们同时执行不同的任务，从而提高程序的执行效率。
- 例如：一个程序需要同时执行`func_a`和`func_b`，如果串行执行，总耗时是两个函数耗时之和；如果使用多线程，可以让两个函数近乎同时执行，总耗时约等于耗时最长的那个函数。

#### 1.3 图示理解

```
hello.py 运行
    │
    ├── 主线程（默认创建）
    │       ├── 执行 func_a
    │       └── 执行 func_b（等 func_a 结束后）
    │
    └── 改进：主线程 + 子线程
            ├── 主线程：执行 func_a
            └── 子线程：同时执行 func_b
```

### 第2部分：多线程完成多任务

#### 2.1 多线程实现步骤

1. **导入线程模块**

   ```python
   import threading
   ```

2. **通过线程类创建线程对象**

   ```python
   线程对象 = threading.Thread(target=任务名)
   ```

3. **启动线程执行任务**

   ```python
   线程对象.start()
   ```

#### 2.2 线程类参数详解

```python
threading.Thread([group [, target [, name [, args [, kwargs]]]]])
```

| 参数   | 说明                                                         |
| ------ | ------------------------------------------------------------ |
| group  | 线程组，目前只能使用`None`                                   |
| target | 执行的目标任务名（函数名，不带括号）                         |
| args   | 以**元组**方式给执行任务传参，元组中元素的顺序必须与目标函数参数的顺序一致 |
| kwargs | 以**字典**方式给执行任务传参，字典的`key`必须与目标函数的参数名一致 |
| name   | 线程名，一般不用设置                                         |
| daemon | 设置是否为守护线程（`True`/`False`）                         |

#### 2.3 无参线程示例

```python
import threading
import time

def coding():
    for i in range(3):
        print("coding...")
        time.sleep(0.2)

def music():
    for i in range(3):
        print("music...")
        time.sleep(0.2)

if __name__ == '__main__':
    coding_thread = threading.Thread(target=coding)
    music_thread = threading.Thread(target=music)
    coding_thread.start()
    music_thread.start()
```

**输出**（可能交替出现）：

```
coding...
music...
coding...
music...
coding...
music...
```

#### 2.4 有参线程示例

**元组传参**：

```python
def coding(num, name):
    for i in range(num):
        print(f"{name}正在编写第{i}行代码...")
        time.sleep(0.2)

coding_thread = threading.Thread(target=coding, args=(3, '小明'))
```

**字典传参**：

```python
def music(count, name):
    for i in range(count):
        print(f"{name}正在听第{i}首歌...")
        time.sleep(0.2)

music_thread = threading.Thread(target=music, kwargs={'name': '小明', 'count': 3})
```

> **注意**：元组传参时，参数的顺序必须与函数定义一致；字典传参时，字典的`key`必须与参数名一致。

### 第3部分：线程的注意点

#### 3.1 线程之间执行是无序的

- **原因**：线程的执行由CPU调度决定，操作系统调度哪个线程，哪个线程就先执行，没有调度的线程不能执行。
- **验证**：创建多个线程，多次运行程序，观察各线程的执行顺序可能不同。
- **与进程类比**：进程之间的执行也是无序的。

```python
import threading
import time

def get_info():
    time.sleep(0.5)
    current_thread = threading.current_thread()
    print(current_thread)

if __name__ == '__main__':
    for i in range(10):
        sub_thread = threading.Thread(target=get_info)
        sub_thread.start()
```

每次运行，打印的线程顺序可能不同。

#### 3.2 主线程会等待所有子线程执行结束再结束

- **现象**：即使主线程的代码已经执行完毕，程序也不会立即退出，而是等待所有子线程完成任务后才退出。
- **示例**：主线程sleep(1)后打印“主线程执行完毕”，但子线程需要2.5秒，程序会等待子线程结束后才真正退出。

**守护主线程（设置子线程为守护线程）**：如果希望主线程退出时，子线程立即销毁不再执行，可以设置守护线程。

```python
# 方式1：创建时设置
work_thread = threading.Thread(target=work, daemon=True)

# 方式2：创建后设置
work_thread = threading.Thread(target=work)
work_thread.setDaemon(True)   # 注意：必须在start()之前调用
```

#### 3.3 线程之间共享全局变量

- **验证**：定义一个全局列表`my_list = []`，一个线程向列表添加数据，另一个线程读取列表。读取线程能够看到添加线程写入的数据。
- **结论**：线程天然共享所属进程的全局变量（与进程不同，进程间默认不共享）。

```python
my_list = []

def write_data():
    for i in range(3):
        my_list.append(i)
    print("write:", my_list)

def read_data():
    print("read:", my_list)
```

#### 3.4 线程共享全局变量可能导致数据错误

- **问题场景**：多个线程同时对同一个全局变量进行“读-改-写”操作。
- **典型例子**：两个线程各自对`g_num`进行100万次`g_num += 1`，期望结果是200万，但实际结果可能小于200万（如100多万）。

**错误原因分析**（以两个线程为例）：

1. 初始`g_num = 0`
2. 线程1读取`g_num`（值为0），此时被调度暂停
3. 线程2读取`g_num`（值仍为0），进行加1，写回`g_num = 1`
4. 线程1恢复，它之前读取的值是0，加1后写回`g_num = 1`
5. 两次加1操作，结果只增加了1 → 数据错误

- **解决方案**：使用**线程同步**（互斥锁），保证同一时刻只有一个线程操作全局变量。

### 第4部分：互斥锁的使用

#### 4.1 互斥锁的概念

- **定义**：互斥锁（Mutex，Mutual Exclusion）是一种用于保护共享资源的锁机制，保证同一时刻只有一个线程可以操作共享数据。
- **原理**：多个线程去“抢”这把锁，抢到的线程获得执行权，操作完成后释放锁；其他未抢到的线程进入等待状态，等待锁被释放后继续抢夺。

#### 4.2 基本API

| 方法               | 说明                                                 |
| ------------------ | ---------------------------------------------------- |
| `threading.Lock()` | 创建一个互斥锁对象                                   |
| `mutex.acquire()`  | 获取锁。如果锁已被占用，则阻塞当前线程，直到锁被释放 |
| `mutex.release()`  | 释放锁。让其他等待的线程可以获取                     |
| `mutex.locked()`   | 检查锁是否被占用，返回`True`或`False`                |

#### 4.3 使用步骤

```python
# 1. 创建锁
mutex = threading.Lock()

# 2. 在操作共享资源前上锁
mutex.acquire()

# 3. 操作共享资源（如修改全局变量）
# ... 临界区代码 ...

# 4. 操作完成后释放锁
mutex.release()
```

#### 4.4 互斥锁解决数据错误示例

```python
import threading

g_num = 0
mutex = threading.Lock()

def sum_num1():
    mutex.acquire()
    global g_num
    for i in range(1000000):
        g_num += 1
    mutex.release()
    print("sum1:", g_num)

def sum_num2():
    mutex.acquire()
    global g_num
    for i in range(1000000):
        g_num += 1
    mutex.release()
    print("sum2:", g_num)

if __name__ == '__main__':
    t1 = threading.Thread(target=sum_num1)
    t2 = threading.Thread(target=sum_num2)
    t1.start()
    t2.start()
```

**结果**：`g_num` 最终等于 2000000（正确）。

> **注意**：如果锁的范围过大（如整个循环都加锁），会导致多线程退化成串行执行，失去并发优势。应尽量缩小临界区。

#### 4.5 关于GIL（全局解释器锁）的补充

- **GIL是什么**：GIL（Global Interpreter Lock）是CPython解释器中的一个**隐式全局互斥锁**。
- **核心作用**：保证同一时刻，只有一个线程执行Python字节码。这意味着即使在多核CPU上，Python的多线程也无法真正**并行**执行计算密集型任务。
- **为什么有GIL**：CPython创始人Guido van Rossum在1990年代初的设计决策，目的是降低性能开销和死锁风险。当时多核CPU未普及，并发主要是I/O密集型任务。如今大量C扩展依赖GIL，强行移除会导致生态崩溃。
- **多线程的适用场景**：
  - **I/O密集型任务**（网络请求、文件读写）：适合多线程，因为I/O操作时会释放GIL。
  - **计算密集型任务**：不适合多线程，应使用`multiprocessing`（多进程）或`asyncio`（异步）。
- **显式锁 vs 隐式锁**：
  - `threading.Lock()` 是**显式**的用户互斥锁。
  - GIL 是**隐式**的解释器级别互斥锁。

#### 4.6 死锁

- **定义**：两个或多个线程互相等待对方释放锁，导致所有相关线程都无法继续执行的现象。
- **原因**：未在合适的地方释放锁。例如：线程1获取了锁A，等待锁B；线程2获取了锁B，等待锁A；双方互不相让。
- **简单示例**（PPT中的案例）：

  ```python
  def sum_num1():
      mutex.acquire()   # 上锁，但从未释放
      for i in range(1000000):
          g_num += 1
      print("g_num1:", g_num)
      # 缺少 mutex.release()
  
  def sum_num2():
      mutex.acquire()   # 永远等不到锁，因为线程1没有释放
      for i in range(1000000):
          g_num += 1
      print("g_num2:", g_num)
  ```

- **结果**：应用程序停止响应，无法继续执行。
- **避免方法**：
  - 确保每个`acquire()`都有对应的`release()`（最好使用`try...finally`或上下文管理器`with mutex:`）。
  - 避免嵌套锁。
  - 使用超时机制。

### 第5部分：进程和线程对比

#### 5.1 关系对比

| 关系           | 说明                                             |
| -------------- | ------------------------------------------------ |
| 线程依附于进程 | 没有进程就没有线程，线程不能独立存在             |
| 进程包含线程   | 一个进程默认提供一条主线程，进程可以创建多个线程 |

#### 5.2 区别对比（详细）

| 对比项       | 进程                                         | 线程                                               |
| ------------ | -------------------------------------------- | -------------------------------------------------- |
| 全局变量共享 | 不共享（默认）                               | 共享（需注意资源竞争）                             |
| 资源开销     | 大（创建进程需要分配独立内存空间）           | 小（共享进程资源）                                 |
| 系统角色     | 操作系统资源分配的基本单位                   | CPU调度的基本单位                                  |
| 独立性       | 进程之间独立运行，一个进程崩溃不影响其他进程 | 线程依存在进程中，一个线程崩溃可能导致整个进程崩溃 |
| 多核利用     | 可以充分利用多核CPU（多进程并行）            | 受GIL限制，不能真正并行执行计算任务                |
| 稳定性       | 强（多进程开发比单进程多线程更稳定）         | 相对较弱                                           |

#### 5.3 优缺点对比

| 类型 | 优点                                               | 缺点                                                         |
| ---- | -------------------------------------------------- | ------------------------------------------------------------ |
| 进程 | 可以用多核，稳定性强（一个进程挂掉不影响其他进程） | 资源开销大，创建和切换速度慢                                 |
| 线程 | 资源开销小，创建和切换速度快，方便共享数据         | 不能使用多核（计算密集型受GIL限制），需要处理资源竞争，稳定性相对弱 |

#### 5.4 总结填空题（答案）

1. 进程和线程都是完成**多任务**的一种方式。
2. 多进程要比多线程消耗的资源**多**，但是多进程开发比单进程多线程开发稳定性要**强**。
3. 多线程可以**共享**全局变量。
4. 线程不能单独执行必须依附在**进程**里面。

## 二、关键知识点速查表

| 知识点      | 关键内容                                                           |
| -------- | -------------------------------------------------------------- |
| 线程概念     | CPU调度基本单位，依附于进程                                                |
| 创建线程     | `threading.Thread(target=func)` + `.start()`                   |
| 传参方式     | `args=(元组)` 或 `kwargs={字典}`                                    |
| 执行顺序     | 无序（由CPU调度决定）                                                   |

| 互斥锁      | `mutex = threading.Lock()` → `.acquire()` → 临界区 → `.release()` |
| 死锁       | 互相等待锁释放，导致程序卡死                                                 |
| GIL      | CPython的全局锁，限制多线程并行计算                                          |
| 进程 vs 线程 | 进程资源开销大、不共享变量、可用多核、稳定性强；线程相反                                   |
