# 机器学习的概论

## 1. 核心定义

**机器学习** 是一种让计算机**从数据中学习规律**，而**不需要为每个步骤明确编程**的方法。

-   **传统编程**： 输入 **规则 + 数据** → 输出 **答案**。
-   **机器学习**： 输入 **答案 + 数据** → 输出 **规则**。

---

## 2. 三个核心要素

-   **数据 (Data)**： 模型的“教材”。没有数据，无法学习。
-   **模型 (Model)**： 用来学习规律的“数学公式”或“网络结构”。
-   **学习算法 (Learning Algorithm)**： 根据数据调整模型参数的“方法”，目的是让模型表现越来越好。

---

## 3. 三种主要类型

| 类型        | 核心思想                 | 数据要求                   | 常见例子                                       |
| :-------- | :------------------- | :--------------------- | :----------------------------------------- |
| **监督学习**  | 从“有标准答案”的例题中学习。      | 输入特征（X）+ 标签（Y，即正确答案）   | 垃圾邮件分类（X=邮件内容，Y=是/否）<br>房价预测（X=面积/地段，Y=房价） |
| **无监督学习** | 探索数据中的“隐藏结构”，没有正确答案。 | 只有输入特征（X），没有标签（Y）。     | 客户分群（X=购买记录，自动分成几类人群）<br>新闻自动聚类            |
| **强化学习**  | 通过“试错”和“奖惩”学习最优策略。   | 智能体(Agent)、环境、动作、奖励信号。 | AI玩游戏（AlphaGo）<br>机器人行走控制                  |

---

## 4. 基本工作流程

1.  **收集数据**： 获取足够的高质量数据。
2.  **数据预处理**： 清洗（去除错误）、归一化（统一范围）、划分（训练集/测试集）。
3.  **选择模型**： 根据问题类型选择算法（如决策树、神经网络）。
4.  **训练模型**： 用**训练集**让模型反复学习，调整内部参数。
5.  **评估模型**： 用**测试集**（模型没见过的数据）检验模型表现。
6.  **调优与部署**： 调整超参数提升性能，最后应用到实际中。

---

## 5. 关键术语速查

-   **特征**： 用来做判断的输入属性（如预测天气时的温度、湿度）。
-   **标签**： 我们希望模型预测的目标值（如天气的“晴/雨”）。
-   **训练集 / 测试集**： 训练集用来学习，测试集用来真实评估。两者不能混淆。
-   **过拟合**： 模型把训练数据背下来了，但对新数据表现很差。**（最常见的问题）**
-   **欠拟合**： 模型连训练数据都没学好，表现很差。

---

## 6. 一句话总结

机器学习就是用**数据**代替**固定规则**，让系统具有**适应性和可扩展性**。它的威力来自数据和算力，它的挑战在于**过拟合**和**数据质量**。

## 关联性、回归、无监督学习、强化学习

### 1. 关联性（关联规则学习）

-   **所属类型**： 无监督学习的一种特殊形式。
-   **核心目标**： 发现数据中 **“如果A发生了，那么B也经常发生”** 这种 **项与项之间的强关联**。
-   **通俗理解**： “啤酒与尿布” —— 买尿布的男性顾客也经常买啤酒。
-   **典型应用**： 购物篮分析（推荐“买了A的人还买了B”）、网页点击流分析。
-   **关键指标**：
    -   **支持度**： A和B同时出现的概率（频率）。
    -   **置信度**： 在A出现的情况下，B出现的条件概率。
    -   **提升度**： 相比随机情况，A对B的出现有多大促进作用（>1表示正相关）。
-   **注意**： 关联性 ≠ 因果性。只是统计上的共现关系。

---

### 2. 回归

-   **所属类型**： **监督学习**。用于预测 **连续数值**。
-   **核心目标**： 找到一个函数 $f(x)$ ，使得它能拟合输入特征 $x$ 与连续输出 $y$ 之间的 **数量关系**。
-   **与分类的区别**：
    -   **分类**： 预测类别（是/否，猫/狗）。输出是离散的。
    -   **回归**： 预测数值（房价，温度，销量）。输出是连续的。
-   **常见方法**：
    -   **线性回归**： 假设关系是直线（ $y = ax + b$ ）。
    -   **多项式回归**： 用曲线拟合。
    -   **逻辑回归**： 名字带回归，但实际是做**二分类**（输出概率，属于广义线性模型）。
-   **评估指标**： 均方误差（MSE）、平均绝对误差（MAE）、R平方（决定系数）。

---

### 3. 无监督学习（Unsupervised Learning）

-   **核心定义**： 数据**没有标签**（只有 $x$，没有 $y$）。模型自己从数据中发现隐藏的结构或模式。
-   **主要任务（三大类）**：

| 任务       | 目标                            | 典型算法                  |
| :------- | :---------------------------- | :-------------------- |
| **聚类**   | 将相似的数据点自动分成若干组（簇）。            | K-Means, DBSCAN, 层次聚类 |
| **降维**   | 在保留主要信息的前提下，减少特征数量（便于可视化或压缩）。 | PCA（主成分分析）, t-SNE     |
| **关联规则** | 找到频繁出现的项集及关联关系。               | Apriori, FP-Growth    |

-   **应用举例**：
    -   客户分群（电商自动划分高、中、低价值客户）。
    -   图像压缩（用降维算法减少存储大小）。
    -   异常检测（自动发现远离所有簇的离群点）。

---

### 4. 强化学习（Reinforcement Learning）

-   **核心定义**： 智能体（Agent）通过 **与环境互动**，试错学习，以**最大化累积奖励**为目标。
-   **关键要素**：

```
智能体 —— 执行动作 ——> 环境
    ↑                    ↓
    +——— 奖励/惩罚 <——— 新状态
```

-   **不是监督学习**： 没有标准答案（标签），只有奖励信号（可能延迟）。
-   **不是无监督学习**： 目标明确（要高分），不是单纯找结构。
-   **核心概念**：
    -   **状态**： 当前环境的描述（如棋局盘面）。
    -   **动作**： 智能体可采取的操作（如落子位置）。
    -   **奖励**： 即时反馈得分（吃到一个金币+1）。
    -   **策略**： 决定“在什么状态下该做什么动作”的规则。
    -   **折扣因子**： 平衡当前奖励 vs 未来奖励的重要性。
-   **典型应用**： AlphaGo、自动驾驶决策、机器人控制、游戏AI（如打《超级玛丽》）。

---

### 5. 一句话对比总结

| 概念 | 有无标签 | 核心问题 | 反馈形式 |
| :--- | :--- | :--- | :--- |
| **回归** | 有（连续数值标签） | 数值是多少？ | 真实值作为损失 |
| **关联规则** | 无（但挖掘项间关系） | 什么和什么常一起出现？ | 无反馈，统计支撑度 |
| **无监督学习** | 无 | 数据内部有什么结构？ | 无反馈 |
| **强化学习** | 无（但有奖励信号） | 如何行动能得最多分？ | 奖励/惩罚（延迟） |

> **记忆小贴士**：  
> - 回归 = 预测数量（画线拟合）  
> - 关联 = 找搭档（A → B）  
> - 无监督 = 自动分类/压缩（无人指导）  
> - 强化 = 训练宠物（做对了给零食）
