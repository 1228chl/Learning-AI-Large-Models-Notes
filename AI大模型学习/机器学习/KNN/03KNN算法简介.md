# KNN 算法简介

## 一、KNN 算法思想与核心概念

### 1.1 算法名称与基本思想

**全称**：K-Nearest Neighbor（K-近邻算法）

**核心思想**：

> 如果一个样本在特征空间中的 **k 个最相似的样本** 中的大多数属于某一个类别，则该样本也属于这个类别。

**深层理解**：

- 这是一种“**物以类聚，人以群分**”的数学表达。

- 算法假设：相似的样本在特征空间中彼此靠近，因此未知样本的类别可以由其附近已知样本的类别来推断。

- 不涉及任何复杂的数学建模，而是直接利用数据本身的分布做决策。

- “近朱者赤，近墨者黑”——通过观察一个样本周围的邻居（最相似的几个样本）的类别，来推断该样本的类别。

**特点**

- 是非参数算法（不对数据分布做任何假设）。

- 是 **惰性学习（Lazy Learning）**：训练阶段只保存数据，不显式学习模型；预测时才进行计算。

- 算法简单，但计算和存储开销大（需保存全部训练数据，预测时需计算所有距离）。

**比喻**（帮助理解，但笔记中可不写）：

> 想知道一个人是什么样的人，就去看他最常交往的 5 个朋友是什么样的人——这就是 KNN 的核心逻辑。

---

### 1.2 样本相似性的本质

**关键问题**：

> 思考：如何确定样本的相似性？

**答案**：

> 样本都是属于同一个任务数据集的。样本在**特征空间中的距离越近，则越相似**。

**深度扩展**：

- **特征空间**：每个样本可以看作高维空间中的一个点，点的坐标由它的特征值决定（例如一个样本有 n 个特征，就是 n 维空间中的一个点）。

- **距离**：两个点之间距离的数值，反映了它们之间的“远近”关系。距离越小，越相似。

- **为什么距离能代表相似性**：在有效特征工程的前提下，特征相似的样本在空间位置上确实会聚集在一起。这是 KNN 有效的前提条件。

---

### 1.3 KNN 解决的问题类型

|问题类型|输出类型|典型应用|决策方式|
|---|---|---|---|
|**分类**|离散类别|手写数字识别、垃圾邮件判断、鸢尾花分类|多数表决（投票）|
|**回归**|连续数值|房价预测、股票价格预测、温度估计|算术平均值|

**扩展**：

- 分类问题中，输出是“红色/蓝色”、“是/否”、“0~9”这类有限集合。
  
- 回归问题中，输出是“300 万”、“25.6 度”这类任意实数。
  
- KNN 是少数能够同时处理分类和回归问题的算法之一。

---

## 二、KNN 算法的核心要素

### 2.1 样本相似性（Similarity）

- **判断依据**：在特征空间中，两个样本的距离越小，则越相似。

- **距离度量**：最常用的是欧氏距离，还有曼哈顿距离、切比雷夫距离、闵可夫斯基距离等。

---

### 2.2 K 值（邻居个数）

- **超参数**：需要人工选择或通过交叉验证自动选择。

- **K 值过小**：模型过于复杂，容易受异常点影响（过拟合）。

- **K 值过大**：模型过于简单，受样本均衡性影响（欠拟合）。

- **极端情况**：K = N（N 为训练样本数），则预测结果总是训练集中最多的类别（忽略所有个体特征）。

---

### 2.3 抉择规则

- **分类**：多数表决（少数服从多数）。

- **回归**：取 K 个邻居的目标值的 **算术平均值**。

---

## 三、KNN 分类与回归的详细流程

### 3.1 KNN 分类流程

1. **计算位置样本到每一个训练样本的距离**（使用选定的距离度量）。
	- 使用欧氏距离或其他距离度量，得到 N 个距离值（N=训练集大小）

2. **将训练样本按距离从小到大升序排列。**
	- 距离越小越靠前，表示越相似

3. **取出距离最近的 K 个训练样本。**
	- 取前 K 个作为“邻居”

4. **多数表决**：统计这 K 个样本中每个类别出现的次数。
	- 统计 K 个邻居中每个类别的出现次数

5. **将未知样本归属到出现次数最多的类别。**
	- 如果有平局（例如 K=4，2 票对 2 票），可随机选或减小 K 为奇数

> 示例：
> 计算《唐人街探案》与已有 9 部电影的特征（如打斗镜头、接吻镜头）的欧氏距离，取 K=5，看 5 部最近邻中动作片多还是爱情片多，决定其类型。

**扩展**：加权投票（常见优化）

- 基本多数表决：每个邻居投票权重相等。
- 加权投票：距离越近的邻居投票权重越大（例如权重 = 1/距离）。这样更靠近的邻居对决策影响更大，通常能提升准确率。

---

### 3.2 KNN 回归流程

1. 计算未知样本到每一个训练样本的距离。
	- 同分类

2. 将训练样本根据距离大小**升序排列**。
	- 同分类

3. 取出距离最近的**K 个训练样本**。
	- 同分类

4. 计算这 K 个样本的**目标值的平均值**。
	- 平均值 = $(y_1 + y_2 + ... + y_K) /K$

5. 将该平均值作为位置样本的预测值。
	- 输出一个连续数值

**扩展**：

- 基本回归：简单平均。
- 加权回归：每个邻居的目标值乘以权重（权重 = 1/距离），再求和后除以总权重。这样更近的样本对预测值影响更大。

**示例**：

- 预测某房子的价格：找到特征最相似的 5 个已售房子，取它们的价格平均值作为预测价格。

---

## 四、K 值的选择——模型复杂度的核心控制

### 4.1 K 值过小（例如 K = 1）

> K 值过小：用较小领域中的训练实例进行预测，容易受到异常点的影响；K 值的减小就意味着整体模型变得复杂，容易发生过拟合。

**深度解析**：

| 维度     | 说明                          |
| ------ | --------------------------- |
| 模型复杂度  | 高（决策边界非常曲折，试图拟合每一个训练点）      |
| 训练集表现  | 非常好（几乎完美，K = 1 时训练集准确率 100% |
| 测试集表现  | 差（对新样本泛化能力弱）                |
| 对噪声敏感度 | **极高**（一个异常点就可能完全改变预测结果）    |
| 决策边界特点 | 局部细节过多，在两类交界处呈现“锯齿状”        |

**举例**：K = 1 时，新样本的类别完全由最近的一个近邻决定。如果这个邻居恰巧是标注错误或测量异常的样本，预测就会错误。

---

### 4.2 K 值过大（例如 K = N）

> K 值过大：用较大邻域中的训练实例进行预测，受到样本均衡的问题；且 K 值的增大就意味着整体的模型变得简单，欠拟合。

**深度解析**：

| 维度     | 说明                       |
| ------ | ------------------------ |
| 模型复杂度  | **低**（决策边界非常平滑，几乎是一条直线）  |
| 训练集表现  | 差（连训练数据都无法准确拟合）          |
| 测试集表现  | 差（偏差大）                   |
| 对噪声敏感度 | **低**（大量邻居的平均会抵消个别异常的影响） |
| 决策边界特点 | 过度平滑，丢失了局部信息             |

**举例**：K = N（N 为训练样本个数），无论输入实例是什么，只会按训练集中最多的类别进行预测，受到样本均衡的影响。

**极端情况 K = N**：

- **分类**：预测结果始终是训练集中出现次数最多的那个类别（忽略样本的所有个体特征）。
- **回归**：预测结果始终是训练集全部目标值的平均值。

---

### 4.3 K 值选择的实践方法

> 如何对 K 超参数进行调优？需要一些方法来寻找这个最合适的 K 值——交叉验证、网格搜索。

**常见方法总结**：

| 方法       | 说明                              | 优点          | 缺点    |
| -------- | ------------------------------- | ----------- | ----- |
| **经验法则** | K 值 取较小的奇数（3,5,7,9,...），常用范围 3~15 | 简单快速        | 不精确   |
| **交叉验证** | 将训练集分成多份，轮流做验证，选平均分数最高的 K        | 可靠，充分利用数据   | 计算量稍大 |
| **网格搜索** | 自动遍历指定的 k 值候选列表，结合交叉验证选出最优        | 自动化，可同时调多参数 | 计算量大  |

**经验补充**：

- K 一般不小于 3（太小易过拟合）。
- K 一般不大于训练集样本数的平方根（大致经验）。
- 对于二分类问题，K 通常取奇数以避免平局。

## 五、距离度量

### 5.1 通用符号定义

**两个 n 维向量**：

$$a = (x_11,x_12,...,x_n)$$

$$b = (x_21,x_22,...,x_{2n})$$

---

### 5.2 欧氏距离**（Euclidean Distance）

> 直观的距离度量方法，两个点在空间中的距离一般都是指欧氏距离。

**公式**：

$$d=\sqrt{\left(x_{11}-x_{21}\right)^2+\left(x_{12}-x_{22_{}}\right)^2+\cdots+\left(x_{1n}-x_{2n}\right)^2}$$

**特点**：

- 最直观、最常用的距离度量。
- 对应几何中的直线距离。
- 对特征的量纲敏感（需要特征预处理）。

**几何意义**：在 n 维空间中两点之间的直线段长度。

---

### 5.3 曼哈顿距离（Manhatten Distance）

> 也称为“城市街区距离”（City Block Distance），曼哈顿城市特点：横平竖直。

**公式**：

$$d=\left|x_{11}-x_{21}\right|+\left|x_{12}-x_{22}\right|+\cdots+\left|x_{1n}-x_{2n}\right|$$

**特点**：

- 各维度差值绝对值的和。
- 等价于只能沿坐标轴方向移动时的最短路径长度。
- 比欧氏距离对异常值的敏感度稍低。

**几何意义**：在城市网格中，从一个十字路口到另一个十字路口需要走过的街道长度。

---

### 5.4 切比雪夫距离（Chebyshev Distance）

> 国际象棋中，国王可以直行、横行、斜行，所以国王走一步可以移动到相邻 8 个方格中的任意一个。国王从格子 $(x1，y1)$ 走到格子 $(x2，y2)$ 最少需要多少步？这个距离就叫切比雪夫距离。

**公式**：

$$d=\max\left(\left|x_{11}-x_{21}\right|,\left|x_{12}-x_{22}\right|,\ldots,\left|x_{1n}-x_{2n}\right|\right)$$

**特点**：

- 取各维度差值绝对值中**最大值**。
- 对应“切比雪夫范数”或“L $\infty$ 范数”。

**几何意义**：棋盘上国王移动的最少步数（可以走对角线）。

---

### 5.5 闵可夫斯基距离（Minkowski Distance）

> 闵氏距离不是一种新的距离度量方式，而是对多个距离度量公式的概括性表述。

**公式**：

$$d=\left(\sum_{i=1}^{n}\left\vert x_{1i}-x_{2i}\right\vert^{p}\right)^{\frac{1}{p}}$$

**参数 p 的含义**：

| p 值  | 对应的距离  |
| ---- | ------ |
| p=1  | 曼哈顿距离  |
| p=2  | 欧氏距离   |
| p->∞ | 切比雪夫距离 |

**意义**：闵可夫斯基距离提供了一个统一的数学框架，通过调整 p 值可以连续地从曼哈顿距离过度到欧氏距离再到切比雪夫距离。

---

### 5.6 距离度量总结表

| 距离名称    | 公式表达式                                                                   | 参数  | 几何直观      |     |
| ------- | ----------------------------------------------------------------------- | --- | --------- | --- |
| 欧氏距离    | $\sqrt{\sum\left(x_{1i}-x_{2i}\right)^2}$                               | 无   | 直线距离      |     |
| 曼哈顿距离   | $\sum\Vert x_{1 i}-x_{2 i}\Vert$                                        | 无   | 网格街道距离    |     |
| 切比雪夫距离  | $\max(\Vert x_{1 i}-x_{2 i}\Vert)$                                      | 无   | 棋盘国王步数    |     |
| 闵可夫斯基距离 | $\left(\sum_{i=1}^{n}\Vert x_{1i}-x_{2i}\Vert^{p}\right)^{\frac{1}{p}}$ | p   | 以上三种的通用形式 |     |

## 六、KNN 的完整算法优缺点分析

### 6.1 优点

| 优点             | 详细说明                                                     |
| -------------- | -------------------------------------------------------- |
| 简单直观           | 算法思想易于理解，无需复杂数学背景即可掌握                                    |
| 无需训练           | 惰性学习，只保存数据，没有显式训练过程（时间复杂度为 0？，不对，训练阶段缺失没有计算，所有的计算都在预测阶段） |
| 非线性            | 能处理复杂的非线性分类/回归问题，决策边界可以非常灵活                              |
| 多类别支持          | 天然支持多分类（而某些二分类算法需要修改才能支持多分类）                             |
| 对异常值不敏感（K 较大时） | K 较大时，个别异常点的影响被附近多数正常点稀释                                 |
| 参数少            | 仅有 K 和距离度是两个主要超参数                                        |

---

### 6.2 缺点

| 缺点       | 详细说明                                | 可能改进的方法                         |
| -------- | ----------------------------------- | ------------------------------- |
| 计算复杂度高   | 预测时需要计算与**所有**训练样本的距离，O(N)，N 为训练集大小 | 使用 KD 树、球树等索引结构；或使用近似最近邻（ANN）算法 |
| 空间复杂度高   | 需要保存**全部**训练数据                      | 可进行数据裁剪（Condensed KNN）或使用原型选择方法 |
| 维度灾难     | 高维空间中距离趋于均匀，“最近”和“最远”的差别变小，KNN 失效   | 先做特征降维（PCA、t-SNE）或特征选择          |
| 对不平衡数据敏感 | 多数类样本数最多，在 K 近邻投票中容易压制少数类           | 使用加权投票（给少数类更高权重）；或过采样/欠采样       |
| 惰性学习的代价  | 虽然“不训练”听起来好，但预测时的高计算量可能不适合实时系统      | 对于实时应用，考虑在预测前预先计算部分结果           |

---

### 6.3 维度灾难的深入解释

**问题描述**：

- 当特征维度 n 增加时，同样大小的数据集在空间中分布变得极其稀疏。
- 在高维空间中，所有样本对之间的距离趋向于非常接近（最远点和最近点差距缩小）。
- KNN 依赖“距离近 = 相似”的假设，当所有距离都差不多时，KNN 就失去了判断依据。

**经验法则**：

- KNN 适合特征数在几十以下的场景。
- 当特征数成百上千时，必须先做降维，否则 KNN 效果会很差。

## 七、KNN 算法的数学本质

### 7.1 从函数逼近角度理解

- KNN 实际上是在用训练样本点对特征空间进行**Voronoi 划分**（泰森多边形）。
- 每个训练样本“负责”它周围的一块区域。
- K = 1 时，决策边界就是 Voronoi 图的边。
- K > 1 时，相当于对多个 Voronoi 细胞进行投票组合。

---

### 7.2 与线性模型的对比

| 维度   | 线性模型               | KNN       |
| ---- | ------------------ | --------- |
| 模型形式 | 显式数学公式（如 $y=wx+b$） | 无显式公式     |
| 参数数量 | 固定（特征数+1）          | 随训练集变化    |
| 训练过程 | 需要优化参数             | 只保存数据     |
| 决策边界 | 线性的                | 非线性的，分段常数 |
| 可解释性 | 较高（权重含义）           | 较低（邻居投票）  |

---
