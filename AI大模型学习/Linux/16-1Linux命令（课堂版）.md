## 一、详细笔记

### 第1部分：计算机概述

#### 1.1 计算机的组成

计算机由两部分组成：**硬件系统** + **软件系统**。

- **硬件系统**：看得见、摸得着的物理部件。
- **软件系统**：看得见、摸不着的程序和数据。

---

#### 1.2 硬件系统（冯·诺依曼体系结构）

从第一台计算机 ENIAC 到当前最先进的计算机，硬件系统设计都采用**冯·诺依曼体系结构**，包含五大部件：

| 部件         | 功能                                         |
| ------------ | -------------------------------------------- |
| **运算器**   | 负责数据的算术运算和逻辑运算，即数据加工处理 |
| **控制器**   | 计算机的中枢神经，根据程序要求协调各部件工作 |
| **内存储器** | 存储正在运行的程序和数据（如内存条）         |
| **外存储器** | 长期存储数据（如硬盘、SSD）                  |
| **输入设备** | 向计算机输入数据（如键盘、鼠标）             |
| **输出设备** | 输出计算机处理结果（如显示器、打印机）       |

> 通常将运算器和控制器合称为 **CPU**。

---

#### 1.3 软件系统

| 分类         | 说明                                               | 示例                           |
| ------------ | -------------------------------------------------- | ------------------------------ |
| **系统软件** | 管理和控制计算机软硬件资源，是所有软件的基础和核心 | 操作系统、编译程序             |
| **应用软件** | 实现用户各种需求                                   | 音乐播放器、电脑游戏、办公软件 |

---

#### 1.4 操作系统（OS）

- **定义**：管理和控制计算机硬件与软件资源的计算机程序，是直接运行在“裸机”上的最基本的系统软件。任何其他软件都必须在操作系统的支持下才能运行。

**常见操作系统**：

| 类别   | 系统                                                | 特点                                         |
| ------ | --------------------------------------------------- | -------------------------------------------- |
| PC端   | Windows（微软）                                     | 用户友好，软件生态丰富，适合家庭、办公、游戏 |
| PC端   | macOS（苹果）                                       | 稳定流畅，与苹果设备协同，适合设计、开发     |
| PC端   | Linux（开源社区）                                   | 开源安全、可定制，适合服务器、嵌入式系统     |
| 移动端 | Android（谷歌）、iOS（苹果）、Harmony（华为鸿蒙）等 | —                                            |

---

### 第2部分：Linux 操作系统

#### 2.1 起源与发展

- **Unix 系统**：较早被广泛使用的计算机操作系统，由 Ken Thompson 在 AT&T 贝尔实验室实现。后续因开源、版权等问题，延伸出不同 Unix 版本。
- **Linux 诞生**：1991 年，芬兰学生 **Linus Torvalds** 发布 Linux 系统第一个版本，遵循 **GPL 协议**（通用公共许可证），**开源免费**。
- **Linux 定位**：类似 Unix 的操作系统，初衷是替代 Unix，在外观和交互上与 Unix 非常类似（但未抄袭源码）。
- **吉祥物**：企鹅。

---

#### 2.2 Linux 的特点

- 开源、免费
- 拥有最为庞大的源码贡献者
- Linux 操作系统本身包括：Linux 内核、系统库、系统程序。**Linux 内核**是最基础的部分。

---

#### 2.3 Linux 发行版

自发布 Linux 内核以来，很多公司在内核基础上构建了自己的操作系统版本，称为 **Linux 发行版**。

| 发行版     | 说明                                                         |
| ---------- | ------------------------------------------------------------ |
| **Redhat** | 全球最大的 Linux 发行厂商，功能全面、稳定。2018 年被 IBM 收购 |
| **Ubuntu** | Linux 桌面操作系统做的最好的发行版                           |
| **CentOS** | 已被 Redhat 收购，但依然免费                                 |
| **Deepin** | 国内做的最好的一款 Linux 发行版                              |

### 第3部分：VMware 虚拟机

#### 3.1 为什么需要虚拟机

学习 Linux 系统需要有一个可用的 Linux 系统环境。但将个人电脑重装为 Linux 不现实（不适合日常办公）。因此，借助**虚拟化技术**，在电脑中虚拟出一台完整的电脑，用于学习 Linux。

---

#### 3.2 什么是虚拟机

通过虚拟化技术，在系统中用软件模拟计算机硬件，并给虚拟硬件安装真实的操作系统，得到的虚拟电脑称为**虚拟机**。

---

#### 3.3 VMware WorkStation

- **作用**：提供虚拟化环境的软件，课程选用 VMware WorkStation。
- **下载地址**：https://www.vmware.com/cn/products/workstation-pro.html

安装完成后，通过 `win + r` 输入 `ncpa.cpl` 可验证网络适配器是否正常。

---

#### 3.4 安装 CentOS 操作系统

1. **下载 CentOS 7.6 ISO 镜像**：
   - 官方：https://vault.centos.org/7.6.1810/isos/x86_64/
   - 直接下载链接：https://vault.centos.org/7.6.1810/isos/x86_64/CentOS-7-x86_64-DVD-1810.iso

2. **在 VMware 中安装**：
   - 打开 VMware → 创建虚拟机 → 选择下载的 ISO 文件 → 设置用户名密码 → 启动自动安装。

3. **安装完成后**，即可进入 Linux 系统体验。

---

#### 3.5 虚拟机网络设置

安装完成后可配置网络（后续课程会讲解固定 IP）。

### 第4部分：FinalShell 客户端

#### 4.1 为什么需要远程客户端

在 VMware 中直接操作 Linux 命令行不方便，主要问题：

- 内容复制/粘贴跨越 VMware 不方便
- 文件上传/下载跨越 VMware 不方便

通过第三方软件 **FinalShell** 远程连接到 Linux 操作系统，可以极大提升操作便利性。

---

#### 4.2 FinalShell 下载与安装

- Windows 版：http://www.hostbuf.com/downloads/finalshell_install.exe
- Mac 版：http://www.hostbuf.com/downloads/finalshell_install.pkg

双击安装，按提示下一步即可。

---

#### 4.3 连接到 Linux 系统

1. **查询 Linux IP 地址**：
   在 Linux 终端中输入 `ip addr` 或 `ifconfig`，查看 IP 地址。

2. **打开 FinalShell**：
   - 点击“连接管理器” → 新建连接
   - 配置：名称（自定义）、主机（Linux IP 地址）、端口（默认 22）、用户名（如 root）、密码
   - 点击“确定”并双击连接

3. **连接成功**后即可通过 FinalShell 操作 Linux。

> **注意**：Linux 虚拟机重启后 IP 可能改变，需要在 FinalShell 中修改连接配置。

### 第5部分：Linux 常用基础命令

#### 5.1 Linux 目录结构

- Linux 的目录结构是一个**树型结构**。
- Windows 系统可以拥有多个盘符（C盘、D盘），**Linux 没有盘符**，只有一个根目录 `/`，所有文件都在它下面。
- 路径描述方式：
  - Linux：使用 `/` 分隔层级，如 `/usr/local/hello.txt`（开头的 `/` 表示根目录）
  - Windows：使用 `\` 分隔层级，如 `D:\data\work\hello.txt`

---

#### 5.2 命令基础格式

```bash
command [-options] [parameter]
```

- `command`：命令本身
- `[-options]`：可选，命令选项，控制行为细节
- `[parameter]`：可选，命令参数，指向目标

> `[]` 表示可选，实际书写时不带方括号。

---

#### 5.3 四大基础命令

| 命令    | 作用                       | 快捷键/补充 |
| ------- | -------------------------- | ----------- |
| `clear` | 清屏                       | `Ctrl + L`  |
| `ls`    | 展示当前目录下所有文件     | —           |
| `cd`    | 切换目录位置               | —           |
| `pwd`   | 打印当前所在位置的绝对路径 | —           |

---

#### 5.4 家目录与特殊路径符

- **家目录（`~`）**：
  - 普通用户的家目录：`/home/用户名`
  - root 用户的家目录：`/root`
- **快速回家**：`cd ~` 或直接 `cd`（`~` 可省略）
- **特殊路径符**：
  - `.`：当前目录，如 `cd .` 或 `cd ./Desktop`
  - `..`：上一级目录，如 `cd ..` 或 `cd ../..`

---

#### 5.5 绝对路径与相对路径

| 类型         | 说明                            | 示例                     |
| ------------ | ------------------------------- | ------------------------ |
| **绝对路径** | 以根目录 `/` 做起点             | `/home/user/file.txt`    |
| **相对路径** | 以当前目录做起点，不以 `/` 开头 | `./file.txt`、`../data/` |

后续课程中常用**相对路径**。

---

#### 5.6 两个实用技巧

- **Tab 键**：自动补全文件/目录名称
- **↑ / ↓ 箭头**：上下翻看历史命令

#### 5.7 ls 命令详解

`ls` 命令用于列出目录下的内容。

| 语法      | 说明                                             |
| --------- | ------------------------------------------------ |
| `ls`      | 平铺形式列出当前目录内容                         |
| `ls -a`   | 显示隐藏文件（以 `.` 开头的文件）                |
| `ls -l`   | 以详细信息显示文件（可简写为 `ll`）              |
| `ls -lh`  | 配合 `-l` 使用，人性化显示文件大小（如 K、M、G） |
| `ls 路径` | 查看指定路径下的文件列表                         |

**示例**：

```bash
ls -la /home    # 查看 /home 目录下所有文件（含隐藏）的详细信息
```

#### 5.8 帮助手册

| 命令          | 说明                   |
| ------------- | ---------------------- |
| `命令 --help` | 查看简单帮助文档       |
| `man 命令`    | 查看详细手册（manual） |

**示例**：

```bash
ls --help
man ls
```

#### 5.9 mkdir —— 创建目录

```bash
mkdir [-p] 目录路径
```

| 用法                     | 说明                                     |
| ------------------------ | ---------------------------------------- |
| `mkdir 目录名`           | 创建单级目录                             |
| `mkdir 目录1 目录2`      | 创建多个同级目录                         |
| `mkdir -p 父目录/子目录` | 自动创建不存在的父目录，用于创建嵌套目录 |

**示例**：

```bash
mkdir test
mkdir a b c
mkdir -p a/b/c
```

#### 5.10 touch —— 创建文件

```bash
touch 文件路径
```

**示例**：

```bash
touch file.txt
touch file1.txt file2.txt
```

#### 5.11 文件内容查看命令

| 命令        | 用途           | 常用选项                                                   |
| ----------- | -------------- | ---------------------------------------------------------- |
| `cat 文件`  | 查看小文件     | `-n` 显示行号                                              |
| `more 文件` | 分页查看大文件 | 空格下一页、`b` 上一页、`q` 退出；`more -10 文件` 每页10行 |
| `head 文件` | 查看文件头部   | `head -20 文件` 显示前20行（默认10行）                     |
| `tail 文件` | 查看文件尾部   | `tail -20 文件` 显示后20行（默认10行）                     |

---

#### 5.12 cp —— 复制

```bash
cp [-r] 源 目标
```

- 不加 `-r`：复制文件
- 加 `-r`：复制文件夹（递归复制）

**示例**：

```bash
cp file.txt /home/user/
cp -r dir1 /home/user/
```

> 复制后源文件依然存在。

---

#### 5.13 mv —— 移动/重命名

```bash
mv 源 目标
```

- 如果目标位置是一个**目录**且存在，则移动文件/文件夹到该目录
- 如果目标位置是一个**不存在的文件名**，则实现重命名

**示例**：

```bash
mv file.txt /home/user/      # 移动文件
mv oldname.txt newname.txt   # 重命名
```

> 移动后源文件不存在。

---

#### 5.14 rm —— 删除

```bash
rm [-r -f] 要删除的文件或文件夹
```

| 选项 | 说明                   |
| ---- | ---------------------- |
| `-r` | 删除文件夹（递归删除） |
| `-f` | 强制删除，不提示确认   |

**示例**：

```bash
rm file.txt
rm -r dir1
rm -rf dir2     # 强制删除，慎用！
```

> **警告**：`rm -rf /` 会删除整个系统，**千万不能执行**。

#### 5.15 echo 与重定向

| 命令          | 说明                               |
| ------------- | ---------------------------------- |
| `echo "内容"` | 打印内容到控制台                   |
| `>`           | 覆盖写入（会覆盖原文件内容）       |
| `>>`          | 追加写入（保留原内容，在末尾追加） |

**示例**：

```bash

echo "World" >> test.txt     # 追加 World
```

#### 5.16 tar —— 打包与压缩

**tar 是 Linux 中最常用的打包/压缩工具**。

##### 普通打包（tar，不压缩）

| 操作 | 语法                            |
| ---- | ------------------------------- |
| 打包 | `tar -cvf 包名.tar 文件或目录`  |
| 解包 | `tar -xvf 包名.tar -C 目标路径` |

**参数说明**：

- `-c`：创建打包文件（打包模式）
- `-x`：解压模式
- `-v`：显示打包/解包过程（verbose）
- `-f`：指定要打包/解包的文件，**`-f` 必须在所有选项最后**
- `-C`：指定解包的目的地（仅解包模式）

##### gzip 压缩（tar.gz）

| 操作 | 语法                                |
| ---- | ----------------------------------- |
| 压缩 | `tar -zcvf 包名.tar.gz 文件或目录`  |
| 解压 | `tar -zxvf 包名.tar.gz -C 目标路径` |

**额外参数**：

- `-z`：使用 gzip 模式（不加 `-z` 为普通 tarball）

**示例**：

```bash
# 打包
tar -cvf myfiles.tar /home/user/*.txt
# 解包到 /tmp 目录
tar -xvf myfiles.tar -C /tmp

# gzip 压缩
tar -zcvf backup.tar.gz /home/user/data
# gzip 解压到当前目录
tar -zxvf backup.tar.gz -C ./
```

### 第6部分：vi/vim 文本编辑器

#### 6.1 简介

- **vi**：Unix 及其衍生系统内置的文本编辑器，具有强大的文本编辑能力。
- **vim**：从 vi 发展出来的增强版（Vi IMproved），支持代码补全、编译、错误跳转等编程功能。
- **核心设计理念**：让程序员的手指始终保持在键盘核心区域，就能完成所有编辑操作。
- **最大挑战**：万物皆命令。

---

#### 6.2 vim 的三种工作模式

```
┌─────────────┐    按 i      ┌─────────────┐
│  命令模式   │ ──────────→ │  输入模式   │
│ (Command)   │ ←────────── │  (Insert)   │
│             │    按 Esc   │             │
└──────┬──────┘             └─────────────┘
       │ 按 :（英文冒号）
       ↓
┌─────────────┐
│ 底线命令模式 │
│ (Last line) │
└─────────────┘
```

| 模式             | 说明                                         | 进入方式         | 退出方式                                     |
| ---------------- | -------------------------------------------- | ---------------- | -------------------------------------------- |
| **命令模式**     | 默认模式，按键被理解为命令，不能自由编辑文本 | 打开 vim 即进入  | 按 `i` 进入输入模式；按 `:` 进入底线命令模式 |
| **输入模式**     | 可自由编辑文件内容                           | 命令模式下按 `i` | 按 `Esc` 退回命令模式                        |
| **底线命令模式** | 用于保存、退出、查找、设置等操作             | 命令模式下按 `:` | 执行命令后自动返回命令模式；或按 `Esc` 取消  |

---

#### 6.3 vim 基础操作

**打开/创建文件**：

```bash
vim 文件名
```

**命令模式常用快捷键**：

| 快捷键       | 作用                               |
| ------------ | ---------------------------------- |
| `i`          | 进入输入模式（在光标前插入）       |
| `I`          | 进入输入模式（在行首插入）         |
| `a`          | 进入输入模式（在光标后插入）       |
| `A`          | 进入输入模式（在行尾插入）         |
| `o`          | 在当前行下方另起一行，进入输入模式 |
| `O`          | 在当前行上方另起一行，进入输入模式 |
| `dd`         | 删除当前行                         |
| `yy`         | 复制当前行                         |
| `p`          | 粘贴到光标下方                     |
| `u`          | 撤销上一步操作                     |
| `Ctrl + r`   | 重做（撤销撤销）                   |
| `gg`         | 跳转到文件开头                     |
| `G`          | 跳转到文件末尾                     |
| `:n` 或 `nG` | 跳转到第 n 行（如 `:10` 或 `10G`） |
| `/关键词`    | 向下搜索关键词                     |
| `?关键词`    | 向上搜索关键词                     |
| `n`          | 查找下一个匹配                     |
| `N`          | 查找上一个匹配                     |

**底线命令模式常用命令**：

| 命令          | 作用                       |
| ------------- | -------------------------- |
| `:w`          | 保存文件                   |
| `:q`          | 退出 vim                   |
| `:wq` 或 `:x` | 保存并退出                 |
| `:q!`         | 强制退出不保存             |
| `:set nu`     | 显示行号（number）         |
| `:set nonu`   | 取消行号                   |
| `:s/旧/新/g`  | 将当前行所有“旧”替换为“新” |
| `:%s/旧/新/g` | 将全文所有“旧”替换为“新”   |

## 三、常见面试/考试题

1. **Linux 的创始人是谁？诞生于哪一年？**  
   林纳斯·托瓦兹（Linus Torvalds），1991 年。

2. **Linux 有哪些常见发行版？**  
   Redhat、Ubuntu、CentOS、Deepin。

3. **Linux 的目录结构和 Windows 有什么不同？**  
   Linux 只有一个根目录 `/`，没有盘符概念；Windows 有多个盘符（C:、D:）。路径分隔符 Linux 用 `/`，Windows 用 `\`。

4. **`ls -a` 和 `ls -l` 分别有什么作用？**  
   `-a` 显示隐藏文件，`-l` 以详细信息显示。

5. **如何查看一个文件的最后 20 行？**  
   `tail -20 文件名`

6. **`cp` 和 `mv` 的区别是什么？**  
   `cp` 复制文件，原文件保留；`mv` 移动文件，原文件被删除。

7. **`rm -rf` 命令有什么风险？**  
   强制递归删除，如果不小心执行 `rm -rf /` 会删除整个系统。

8. **tar 命令中 `-z`、`-c`、`-x`、`-v`、`-f`、`-C` 分别代表什么？**  
   - `-z`：gzip 压缩/解压
   - `-c`：创建打包文件
   - `-x`：解包
   - `-v`：显示过程
   - `-f`：指定文件（必须最后）
   - `-C`：指定目标目录

9. **vim 有哪三种工作模式？如何切换？**  
   命令模式（默认）、输入模式（按 `i`）、底线命令模式（按 `:`）。按 `Esc` 退回命令模式。

10. **在 vim 中如何不保存强制退出？**  
    `:q!`
